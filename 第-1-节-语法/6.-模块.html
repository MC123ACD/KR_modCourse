6. 模块

6.1 什么是模块？

模块就是一个 Lua 文件，它被另一个 Lua 文件加载后，返回一个表（或其他值）。你可以把模块想象成一个 “工具箱” —— 里面放着各种相关的工具（函数）和零件（变量），需要的时候打开箱子拿一个出来用就行。

模块的主要作用：

  1. 分类整理：把相关的函数和变量放在同一个模块里，避免所有代码都塞进一个文件。
  2. 代码复用：写好的模块可以在多个项目中重复使用，不用复制粘贴。
  3. 避免命名冲突：每个模块有自己独立的命名空间，不同模块里的同名函数不会打架。

假设你在开发一个游戏，如果把所有代码都写在一个文件里，很快就会变成这样：

-- main.lua（反面教材：所有东西挤在一起）
-- 游戏配置
local game_config = { width = 800, height = 600 }

-- 玩家相关
local player_name = "英雄"
local player_health = 100
local function take_damage(amount) ... end
local function heal(amount) ... end

-- 怪物相关
local monsters = {}
local function create_monster(name, health) ... end
local function attack_monster(index, damage) ... end

-- UI 相关
local function show_hud() ... end

-- 游戏主循环
local function game_loop() ... end
-- 问题：
-- 1. 几千行代码混在一起，想找个函数都要翻半天
-- 2. 变量名一不小心就会重复（比如都叫 `name`）
-- 3. 想在其他游戏里复用“玩家系统”？只能复制粘贴
-- 4. 多人协作时，改同一文件天天冲突```

模块就是为了解决这些问题而生的。

6.2 创建你的第一个模块

新建一个文件 math_utils.lua：

-- math_utils.lua
-- 1. 创建模块表（相当于工具箱）
local math_utils = {}

-- 2. 模块常量
math_utils.PI = 3.1415926535898

-- 3. 把函数放进工具箱
function math_utils.add(a, b)
    local result = a + b
    print(string.format("add: %d + %d = %d", a, b, result))

    return result
end

function math_utils.subtract(a, b)
    local result = a - b
    print(string.format("subtract: %d - %d = %d", a, b, result))

    return result
end

function math_utils.multiply(a, b)
    local result = a * b
    print(string.format("multiply: %d * %d = %d", a, b, result))

    return result
end

function math_utils.divide(a, b)
    if b == 0 then
        print("divide: 除数不能为 0")
        return nil
    end

    local result = a / b
    print(string.format("divide: %d / %d = %d", a, b, result))

    return result
end

-- 4. 内部辅助函数（不暴露给外部，外面无法直接调用）
local function is_number(value)
    return type(value) == "number"
end

-- 5. 更复杂的函数：平均数
function math_utils.average(...)
    local numbers = { ... }
    local sum, count = 0, 0

    for i = 1, #numbers do
        if is_number(numbers[i]) then
            sum = sum + numbers[i]
            count = count + 1
        end
    end

    if count == 0 then
        print("average: 没有有效数字")
        return nil
    end

    local result = sum / count
    print(string.format("average: 平均数 = %f", result))

    return result
end

-- 6. 最后必须返回这个表（工具箱）
return math_utils

要点：

  • 模块通常返回一个表，表里放着对外提供的函数和变量。
  • 不放在返回表里的变量（比如 is_number）就是“私有”的，外部无法访问。
  • return 之后的代码不会执行，所以一定要放在最后一行。

6.2.2 使用模块

新建另一个文件（比如 test.lua），用 require 加载模块：

-- test.lua
-- require 会找到并执行 math_utils.lua，然后返回模块表
local math_utils = require("math_utils")

math_utils.add(5, 3)			--> add: 5 + 3 = 8
math_utils.subtract(10, 4)		--> subtract: 10 - 4 = 6
math_utils.multiply(6, 7)		--> multiply: 6 * 7 = 42
math_utils.divide(10, 2)		--> divide: 10 / 2 = 5

print("π ≈ " .. math_utils.PI)	--> π ≈ 3.1415926535898

-- 尝试访问内部函数（失败，因为它是私有的）
-- math_utils.is_number(123)  -- 报错：nil
提示

如果模块忘记返回表，math_utils.add 尝试访问会报错 attempt to index local v (a boolean value)

6.2.3 模块加载机制

require("模块名") 做了这几件事：

  1. 检查 package.loaded 表中是否已经加载过该模块。
  2. 如果已经加载过，直接返回缓存的结果（不会重复执行模块文件）。
  3. 如果没有，就在 package.path 指定的路径中查找文件（比如 math_utils.lua）。
  4. 加载并执行该文件，把 return 的值存入缓存，并返回这个值。
-- require 的缓存特性
local m1 = require("math_utils")
local m2 = require("math_utils")
print(m1 == m2)   --> true（同一个表）

-- 如果想强制重新加载，可以清除缓存
package.loaded.math_utils = nil
local m3 = require("math_utils")
print(m1 == m3)   --> false（重新执行，生成了新表）
信息

手动清除缓存再 require，即可做到重新加载。

6.3 模块的高级特性

6.3.1 模块的多种返回方式

-- 方式1：返回表（最常用）
local M = {}
M.version = "1.0"
return M

-- 方式2：返回函数（工厂模式）
return function(config)
    return {config = config, run = function() print("运行") end}
end
-- 使用时：
-- local create = require("mod")
-- local obj = create({...})

-- 方式3：直接修改全局变量（不推荐，会污染全局空间）
-- 但某些遗留代码中可能见到

6.3.2 模块环境

Lua 中所有全局变量都存在一个叫 _G 的全局表里。每个模块默认也能访问 _G（比如 print、math 都是 _G 里的）。

如果你想创建一个“干净”的模块，不让它访问外部全局变量（除了你明确给的），可以修改模块的环境（setfenv）。

提示

开发中很少需要手动改环境，了解即可。

-- 创建一个只有 print 可用的环境
local sandbox = {print = print}

-- 编译模块代码（字符串形式）
local chunk = loadstring([[
    local M = {}
    
    function M.say()
        print("Hello from sandbox")
        -- 其他 Lua 自带函数不可用
    end
    
    return M
]])

setfenv(chunk, sandbox)   -- 设置环境
local M = chunk()         -- 执行
M.say()                   --> Hello from sandbox

6.3.3 模块之间的依赖

一个模块可以依赖其他模块。

-- logger.lua
local logger = {}

function logger.info(msg)
	print("[INFO] " .. msg)
end

function logger.error(msg)
	print("[ERROR] " .. msg)
end

return logger

-- database.lua（依赖 logger）
local logger = require("logger")
local db = {}

function db.connect(conn_str)
    logger.info("连接数据库: " .. conn_str)
    return {connected = true}
end

return db

-- main.lua
local db = require("database")
db.connect("localhost")
警告

避免循环依赖：如果 A 依赖 B，B 又依赖 A，会导致加载失败。设计模块时应保持单向依赖。

6.4 Lua 标准库模块

Lua 自带了一些非常有用的模块，不需要额外安装，直接 require 或直接使用（有些是全局的）。

6.4.1 math 数学库

print(math.abs(-10))       --> 10
print(math.sqrt(16))       --> 4
print(math.random(1, 100)) --> 随机数
print(math.pi)             --> 3.1415926535898

6.4.2 string 字符串库

print(string.upper("hello"))   --> HELLO
print(string.reverse("world")) --> dlrow
print(string.format("%.2f", 3.1415)) --> 3.14

6.4.3 table 表库

local t = {1, 2, 3}
table.insert(t, 4)     -- 插入
table.remove(t, 2)     -- 删除
table.sort(t)          -- 排序

6.4.4 os 操作系统库

print(os.date("%Y-%m-%d %H:%M:%S"))  --> 当前时间
print(os.time())                     --> 时间戳
os.execute("echo Hello")             -- 执行系统命令（谨慎使用）

6.4.5 io 文件操作库

-- 写文件
local f = io.open("test.txt", "w")
f:write("Hello\n")
f:close()

-- 读文件
local f = io.open("test.txt", "r")
local content = f:read("*all")
f:close()
print(content)

6.4.6 debug 调试库

debug 库主要用于调试，在生产代码中应谨慎使用

-- debug.getinfo 获取函数信息
local function test_function(x, y)
    local sum = x + y
    return sum
end

local info = debug.getinfo(test_function)
print("函数信息:")
print("  定义位置: " .. info.source)
print("  参数数量: " .. info.nparams)

--[[ 输出
函数信息:
  定义位置: test.lua
  参数数量: 2
]]

6.5 使用 LuaRocks 安装库

LuaRocks 是 Lua 的包管理器（类似于 Python 的 pip）。你可以用它来安装别人写好的模块。

LuaRocks 官网：
LuaRocks - The Lua package manager

6.5.1 在终端安装库

luarocks install 库名


以上命令会将 Lua 库安装到 C:/Users/用户名/AppData/Local/VirtualStore/Program Files (x86)/Lua/5.1/lua
由于服务器位于国外，下载较慢耐心等待即可

6.5.2 使用

-- 以 dkjson 库为例
-- 需要确保安装此库： luarocks install dkjson
local dkjson = require("dkjson")

-- 将 lua 表编译为 json
print(dkjson.encode({foo = "bar", baz = 123}))
-- 输出 {"baz":123,"foo":"bar"}

6.6 练习

练习 1：日志模块

创建一个调试用的日志模块 log.lua。

要求：

  1. 支持不同级别：error, warn, info, debug
  2. 支持格式化字符串（类似 string.format）
  3. 提供一个函数，可以设置当前输出的最低级别（低于该级别的不输出）。

示例：

local log = require("log")
log:set_level(log.LEVEL_NAMES.INFO)  -- 设置日志级别为 INFO

log:error("无法加载模块 %s", "log")	--> [ERROR]: 无法加载模块 log
log:info("加载模块中...")				--> [INFO]: 加载模块中...
log:debug("造成伤害：%s", 100)		--> （不打印）
log:debug("恢复血量：%s", 100)		--> （不打印）

练习 2：实体数据库模块

创建一个实体数据库模块 entity_db.lua，用于管理游戏中的玩家、怪物等。每个实体是一个表，包含一个唯一的 id 字段和其他自定义属性。

要求：

  1. 实现创建实体函数：create(entity_data)
    • 自动为实体分配一个唯一的整数 ID（从 1 开始递增）。
    • 将实体数据存储到内部表中。
    • 返回分配后的实体（包含自动生成的 id 字段）。
    • 创建后使用练习 1 的日志库打印信息。
  2. 实现根据 ID 获取实体函数：get(id)
    • 返回指定 ID 的实体表；如果不存在则返回 nil。
  3. 实体表中提供一些函数：
    • 可以对实体造成伤害的函数，如果没有找到目标，使用日志库打印错误。
    • 打印实体信息的函数。

示例：

local entity_db = require("entity_db")

-- 创建实体
local player = entity_db.create({
	name = "小明", 
	type = "player", 
	hp = 100,
})		--> [INFO]: 创建实体：小明(ID: 1)

local monster = entity_db.create({
	name = "哥布林", 
	type = "monster", 
	hp = 50, 
	attack = 10,
})		--> [INFO]: 创建实体：哥布林(ID: 2)

-- 获取实体：玩家
local p = entity_db.get(1)
-- -- 获取实体：怪物
local m = entity_db.get(2)

-- 对玩家造成伤害
m:take_damage(p)	--> [INFO]: 造成伤害：10，目标：小明(ID: 1)，来源: 哥布林(ID: 2)

-- 输出实体信息
p:show_info()
--[[ 输出
========实体信息========
name：小明
type：player
hp：90
]]
