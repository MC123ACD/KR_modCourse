3. 函数

函数就像是一个小工具——你给它一些输入（参数），它帮你完成某个特定任务，然后可能返回一个结果（返回值）。通过函数，我们可以把重复的代码封装起来，随时调用，让程序更简洁、更易维护。

3.1 函数的基本概念

如果让你打印多个玩家信息，没有函数就得写很多多余代码，并且后续如果想打印其他信息这些代码都要修改，有了函数只需要调用函数即可，修改时只需要修改这一个函数便能同步到所有地方。

-- 重复打印多个玩家信息，代码冗长
local player1_name = "小明"
local player1_level = 10
local player1_health = 100
print("姓名：" .. player1_name)
print("等级：" .. player1_level)
print("生命值：" .. player1_health)

local player2_name = "小红"
local player2_level = 8
local player2_health = 80
print("姓名：" .. player2_name)
print("等级：" .. player2_level)
print("生命值：" .. player2_health)

local player3_name = "小刚"
local player3_level = 12
local player3_health = 120
print("姓名：" .. player3_name)
print("等级：" .. player3_level)
print("生命值：" .. player3_health)

3.1.1 函数的语法

-- 定义函数
local function 函数名(参数1, 参数2, ...)
    -- 函数体
    return 返回值   -- 可选
end

-- 调用函数
函数名(参数1, 参数2, ...)

示例：简单的问候函数

-- 定义函数
local function greet()
    print("你好！欢迎来到Lua世界！")
end

-- 调用函数
greet()   --> 你好！欢迎来到Lua世界！
greet()   --> 可以多次调用
greet()

3.2 参数

想象一下，你有一个榨汁机。如果没有参数，这个榨汁机只能榨同一种水果——比如永远只榨苹果。那它只能叫“苹果榨汁机”，不够灵活，如果想榨另一种水果就需要一台新榨汁机。

参数就是让函数变得灵活的关键。它允许你把“输入”传给函数，让同一个函数处理不同的数据。

-- 一个“固定”的问候函数
local function say_hello()
    print("你好，小明！")
end

say_hello()   --> 你好，小明！
say_hello()   --> 你好，小明！（永远只能跟小明打招呼）

-- 带参数的问候函数
local function say_hello(name)
    print("你好，" .. name .. "！")
end

say_hello("小明")   --> 你好，小明！
say_hello("小红")   --> 你好，小红！
say_hello("小刚")   --> 你好，小刚！
信息

函数的参数是个局部变量。

示例：计算伤害

local function calculate_damage(attack, defense, multiplier)
	multiplier = multiplier or 1  -- multiplier 可选，默认为 1
	
    local damage = attack - defense
    
    -- 避免负数伤害回血
    if damage < 0 then
        damage = 0
    end
    
    damage = damage * multiplier
    
    print("造成伤害：" .. damage)
end

-- 测试伤害计算
calculate_damage(50, 30, 1.5)  -- 攻击 50，防御 30，倍率 1.5
calculate_damage(40, 45, 1.0)  -- 攻击 40，防御 45
calculate_damage(100, 20)      -- multiplier 未提供，使用默认值 1

--[[
造成伤害：30 ((50 - 30) * 1.5)
造成伤害：0 ((40 - 45)为负数，设为 0)
造成伤害：80 (100 - 20)
]]

3.2.1 变长参数

有时你不知道函数会收到多少个参数，可以用 ... 表示“剩余所有参数”。

-- 求任意多个数的和
local function sum(...)
    local numbers = {...}   -- 把参数收集到一个表里
    local total = 0
    
    -- # 用于获取有几个数字
    for i = 1, #numbers do
        total = total + numbers[i]
    end
    
    print(total)
end

sum(1, 2)          --> 3
sum(1, 2, 3, 4, 5) --> 15
sum()              --> 0

3.3 返回值

再想想榨汁机：你把水果放进去（参数），它帮你榨好，然后把果汁递出来给你。这个“递出来”的动作，就是返回值。

没有返回值的函数，只能自己把结果打印出来，但其他代码无法使用这个结果。

local function add_and_print(a, b)
    local result = a + b
    
    print("结果是：" .. result)
end

-- 调用函数，它打印了结果，但我们无法把这个结果拿去继续计算
add_and_print(5, 3)   --> 结果是：8

-- 下面这样不行！因为函数没有返回任何东西
local x = add_and_print(10, 20)   -- 这行会打印"结果是：30"
print(x)   --> nil（x 是空的，因为函数没有返回值）

local function add(a, b)
    local result = a + b
    
    return result   -- 把结果“递”出去
end

-- 可以拿到结果继续使用
local sum = add(5, 3)
print("总和是：" .. sum)   --> 总和是：8
-- 还可以嵌套使用
local total = add(10, add(5, 3))  -- 先算 5 + 3 = 8，再算 10 + 8 = 18
print(total)   --> 18
-- 还可以作为其他函数的参数
print(add(100, 200))   --> 300
信息

返回值后面的代码不会被执行

local function is_odd(number)
    return number % 2 == 1
    
    print("这里不会被执行")
end

3.3.1 返回函数

函数也可以返回另一个函数。

local function create_multiplier(factor)
    return function(x)
        return x * factor
    end
end

local double = create_multiplier(2)
local triple = create_multiplier(3)

print(double(5))   --> 10
print(triple(5))   --> 15

3.3.2 卫语句：让代码更清晰

“卫语句”是一种编程技巧：在函数开头先检查所有异常情况，不符合条件就提前返回，避免层层嵌套。

-- 嵌套过深的代码
local function check_level_unlock(player_level, level_mode, unlocked)
    if player_level >= 1 then
        if unlocked then
            if level_mode == "night" then
                print("进入夜晚森林")
            elseif level_mode == "day" then
                print("进入白天森林")
            else
                return "未知模式"
            end
        else
            return "未解锁"
        end
    else
        return "等级不足"
    end
end

-- 用卫语句改写
local function check_level_unlock(player_level, level_mode, unlocked)
    if player_level < 1 then
        return "等级不足"
    end
    
    if not unlocked then
        return "未解锁"
    end
    
    if level_mode == "night" then
        print("进入夜晚森林")
    elseif level_mode == "day" then
        print("进入白天森林")
    else
        return "未知模式"
    end
end

3.4 匿名函数

匿名函数就是没有名字的函数。它像是一个“临时工” —— 需要的时候当场定义，用完就不再保留。

为什么需要匿名函数？

很多场景下，函数只用一次，专门给它起个名字反而多余，匿名函数可以让代码更紧凑、更直观。

-- 基本匿名函数
local add = function(a, b)
    return a + b
end

print("5 + 3 = " .. add(5, 3))  -- 8

-- 立即调用的匿名函数
local result = (function(x)
    return x * x
end)(5)

print("5 的平方：" .. result)  --> 5 的平方：25

3.5 闭包

闭包是一个函数加上它创建时能访问的所有外部变量。闭包可以“记住”这些变量，即使外部函数已经执行完毕。

示例：计数器

local function make_counter()
    local count = 0   -- 这个变量会被闭包 “记住”
    
    return function()
        count = count + 1
        return count
    end
end

local counter1 = make_counter()
local counter2 = make_counter()

print(counter1())  --> 1
print(counter1())  --> 2

print(counter2())  --> 1  （独立的计数器）

3.6 递归

递归就是函数调用自己。递归必须有一个终止条件，否则会无限循环导致栈溢出。

示例：阶乘

local function factorial(n)
    if n <= 1 then
        return 1
    end
    
    return n * factorial(n - 1)
end

print(factorial(5))   --> 120
警告

递归如果层数太多（比如上千层），可能会导致性能问题或栈溢出，对于大数计算建议改用循环。

3.7 文档注释

Lua 的注释是 --。在函数定义前加上 --- 开头的注释，并配合 @param 和 @return，可以在编辑器中显示函数签名。使函数更容易理解和维护。

---对某人说 hello
---@param name string 名称
---@return string 完整文本
local function say_hello(name)
    local text = "hello " .. name
    
    print(text)
    
    return text
end

say_hello("bob")
-- 鼠标悬停在 say_hello 上会显示参数和返回值说明

3.8 协程

协程是 Lua 中一种可以暂停和恢复的函数。普通函数一旦开始执行，就必须一直运行到 return 结束；而协程可以在中途主动“让出”（yield），把控制权交还给调用者，稍后再从让出的地方继续执行。

你可以把协程想象成一个可以随时保存进度的游戏：玩到一半按暂停，去干别的事，回来之后从刚才暂停的地方继续玩。

3.8.1 为什么需要协程？

假设你要写一个游戏，敌人需要不断移动。如果用普通函数，可能会写成死循环：

-- 普通函数：一旦进入就出不来了
local function enemy_ai()
    while true do
        -- 敌人移动逻辑
        print("敌人移动一步")
    end
end

enemy_ai()  -- 程序会永远卡在这里，后面的代码无法执行
print("这行永远不会打印")

这种写法会阻塞整个程序，无法做其他事情（比如同时控制多个敌人、响应玩家输入）。协程可以解决这个问题：让敌人每移动一步就暂停，让其他代码可以继续运行，下次再恢复移动下一步。

信息

协程通常使用较少，可以简单了解一下。

3.8.2 协程四种状态

  1. suspended：挂起（创建后或让出后）。
  2. running：运行中。
  3. normal：正常（恢复其他协程时）。
  4. dead: 结束（函数执行完毕）。

3.8.3 协程的基础函数

1. coroutine.create - 创建协程

coroutine.create(函数) -> 协程

2. coroutine.resume - 重启协程

-- 成功时
coroutine.resume(协程, 传递的参数...) -> true, 函数返回值|让出时传递的参数

-- 失败时
coroutine.resume(协程, 传递的参数...) -> false, 错误信息

3. coroutine.yield - 协程让出

coroutine.yield(参数...) -> 传递的参数

4. coroutine.status - 获取协程状态

coroutine.status(协程) -> "dead"|"normal"|"running"|"suspended"

示例：

local function status_example()
	print("  协程内部状态: " .. coroutine.status(coroutine.running()))
	coroutine.yield()
	print("  恢复执行")
end

local co = coroutine.create(status_example)
print("  创建后状态: " .. coroutine.status(co))

coroutine.resume(co)
print("  第一次 yield 后状态: " .. coroutine.status(co))
coroutine.resume(co)
print("  完成后状态: " .. coroutine.status(co))

--[[ 输出
 创建后状态: suspended
 协程内部状态: running
 第一次 yield 后状态: suspended
 恢复执行
 完成后状态: dead
]]

5. coroutine.wrap - 包装函数

创建一个新协程。返回一个函数，每次调用该函数都会运行该协程。

coroutine.wrap(函数) -> 包装后的函数

示例：

local function counter(max)
	for i = 1, max do
		print("  计数: " .. i)
		coroutine.yield(i)
	end
	
	return "计数完成"
end

-- wrap 返回一个函数，而不是协程
local count_func = coroutine.wrap(counter)

print("  第一次调用:")
local result1 = count_func(5)	-- 相当于 resume

print("  第二次调用:")
local result2 = count_func()

--[[ 输出
  第一次调用:
  计数: 1
  第二次调用:
  计数: 2
]]

6. coroutine.running - 获取当前运行的协程

coroutine.running() -> 当前协程
  • 如果在主线程调用将会返回 nil

示例：

local function get_current_coroutine()
    local running_co = coroutine.running()
    print("  当前协程: " .. tostring(running_co))
    
    if not running_co then
        print("  在主线程中运行")
    else
        print("  在协程中运行")
    end
end

print("  直接调用函数（在主线程）:")
get_current_coroutine()

print("\n  在协程中调用:")
local co_test = coroutine.create(get_current_coroutine)
coroutine.resume(co_test)
--[[ 输出
 直接调用函数（在主线程）:
 当前协程: nil
 在主线程中运行

 在协程中调用:
 当前协程: thread: 0x00d51d78
 在协程中运行
--]]

3.8.4 参数传递

协程向外传递参数是通过 coroutine.yield 挂起时传递给 coroutine.resume。

local function parameter_example(name, count)
    print("  接收参数: name=" .. name .. ", count=" .. count)
    
    for i = 1, count do
        print("  处理第" .. i .. "项")
        local received = coroutine.yield("进度: " .. i .. "/" .. count)
        
        if received then
            print("  收到外部数据: " .. received)
        end
    end
    
    return "处理完成: " .. name
end

local co_param = coroutine.create(parameter_example)

-- 第一次resume传递参数
print("  第一次resume:")
local _, progress = coroutine.resume(co_param, "数据导入", 3)
print(progress)

-- 后续resume传递数据给yield
print("\n  第二次resume（带数据）:")
_, progress = coroutine.resume(co_param, "继续处理")
print(progress)

print("\n  第三次resume（带数据）:")
_, progress = coroutine.resume(co_param, "最后一步")
print(progress)

print("\n  第四次resume（获取最终结果）:")
_, progress = coroutine.resume(co_param)
print(progress)
--[[ 输出
第一次resume:
接收参数: name=数据导入, count=3
进度: 1/3

第二次resume（带数据）:
收到外部数据: 继续处理
进度: 2/3

第三次resume（带数据）:
收到外部数据: 最后一步
进度: 3/3

第四次resume（获取最终结果）:
处理完成: 数据导入
--]]

练习 1：计算器函数库

实现一个计算器函数库：

  1. 实现基础四则运算：可以打印结果、处理除数为 0 的情况并返回值。
  2. 实现计算斐波那契数列的递归函数：计算数列中第 n 项的值。（0，1，1，2... 从第 3 项开始，每一项等于前两项之和）。

示例：

local result
-- 加
result = add(5, 3)
print("上次计算结果：" .. result)
-- 减
result = subtract(10, 4)
print("上次计算结果：" .. result)
-- 乘
result = multiply(6, 7)
print("上次计算结果：" .. result)
-- 除
result = divide(10, 2)
print("上次计算结果：" .. result)
-- 除（错误）
result = divide(10, 0)

-- 斐波那契数列
print("斐波那契数列第 0 - 9 项：")
for i = 0, 9 do
    print(fib(i))
end

练习 2：创建游戏角色生成器

实现 create_character_generator(class_name) 函数：

  • 返回一个函数，每次调用返回该职业的新角色
  • 角色属性根据职业不同：战士（生命 120，攻击 15），法师（生命 80，攻击 25），盗贼（生命 90，攻击 20）

local warrior_gen = create_character_generator("战士")

练习 3：使用协程管理多个敌人

协程管理两个敌人，每个敌人拥有独立的协程，通过主循环控制协程的事件响应。

  1. 协程行为：
    • 每个敌人的协程内部无限循环，等待外部发送事件（"damage" 或 "move"）。
    • 收到 "damage" 事件时扣除对应血量，打印受到伤害信息；若血量 ≤ 0 则打印死亡信息并退出协程。
    • 收到 "move" 事件时打印移动一步的信息。
  2. 主循环逻辑：
    • 只要还有存活的敌人，重复执行以下步骤：
      1. 对所有存活的敌人依次发送 "damage" 事件。
      2. 对所有存活的敌人依次发送 "move" 事件。
    • 所有敌人死亡后，打印 "所有敌人全部死亡终止循环"。

示例：

--[[ 输出
敌人 1：移动一步
敌人 2：移动一步
敌人 1：受到伤害 50
敌人 2：受到伤害 40
敌人 1：移动一步
敌人 2：移动一步
敌人 1：受到伤害 50
敌人 1：死亡
敌人 2：受到伤害 40
敌人 2：移动一步
敌人 2：受到伤害 40
敌人 2：死亡
所有敌人全部死亡终止循环
]]
