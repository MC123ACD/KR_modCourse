环境配置

一、安装 VS Code：

二、安装 Lua 扩展

  1. 点击扩展 → 搜索 Lua → 安装 1.7 M 下载量的。
  2. 搜索 Local Lua Debugger → 安装

如果嫌这个 Lua 扩展卡顿，可以安装上面的 Lua，缺点就是无中文翻译

三、安装 Love 2d 引擎

  1. 下载资源里的 love 安装包 → 安装

  2. 按 win + r → 输入 sysdm.cpl

  3. 点击高级 → 环境变量 → 编辑 path 系统变量

  1. 增加 love 的安装路径（图片路径仅供参考，请填写你的安装目录）

  2. 重启电脑 → 按 win + r 输入 cmd → 终端输入 love + 回车，验证是否出现窗口，如果出现窗口则安装成功

四、打开游戏文件夹

  1. 将要修改的游戏复制到一个文件夹内（可以自己创建） → 解压游戏
    注意：路径不要有中文

推荐放在 D 盘
  1. 将解压出来的文件与游戏本体放到一起，如图二所示
  1. 将文件夹增加到 VScode 的工作区（打开文件夹）

五、配置调试启动文件

  1. 点击运行和调试 → 创建 launch.json 文件 → 调试器选择 Local Lua Debugger
  1. 用以下代码替换启动文件 → 保存
{
	"version": "0.2.0",

	"configurations": [
		{
			"type": "lua-local",
			"request": "launch",
			"name": "Debug",
			"program": {
				"command": "love"
			},
			"args": [
				".",
				"debug"
			]
		},
		{
			"type": "lua-local",
			"request": "launch",
			"name": "Release",
			"program": {
				"command": "love"
			},
			"args": [
				"."
			]
		}
	]
}
  1. 反编译 main → 向 main 增加一块代码 → 保存
    增加到顶部：
if arg[2] == "debug" then
	require("lldebugger").start()
end

六、开启游戏的调试

  1. 打开 version → 将 build 改为 true

六、测试

  1. 打开运行与调试 → 选择模式：Debug 为调试模式（注调试模式会对性能造成较大影响）
    Release 为正常模式（也就是禁用调试功能）

  2. 打一些断点（因为现在是测试所以尽量别打到函数内）

  3. 运行
    注意：运行需要打开 steam（可关闭验证，不打算教防止盗版泛滥）

  4. 一切正常下游戏会在断点处暂停，同时可以看到变量、堆栈

完成后注意一下存档位置会在 C:\Users\你的用户名称\AppData\Roaming\LOVE 里面

一、手动触发断点

设置后可按 0 手动触发断点
断点触发后可以直接右键修改变量的值、以及在调试控制台执行代码（必须使用调试模式运行）

  1. 找到 main 的 love.keypressed 函数（直接搜索就行）
  2. 函数内加块代码，同时增加断点
if key == "0" then
	print("Break-point")
end

二、监视变量

注意：只有触发断点时监视才可用

变量
描述
game.game_gui.selected_entity 被鼠标点击选择的单位的表会显示在内
game.store 临时的表等（创建的单位等都在里面）
game.game_gui 游戏 GUI（进入关卡后的技能、金币、生命等）
screen_map 地图的 UI 与 GUI 等（英雄殿堂、升级等）

三、字体

  1. 打开设置（左下角）
  2. 点击文本编辑器 → 字体 → 输入 JetBrains Mono （请确保已安装）
