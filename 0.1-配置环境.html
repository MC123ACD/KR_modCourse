二、创建工作区文件夹

  1. 将网盘的整合包下载并解压
    解压工具见 0.2 工具 > 压缩软件

链接：

工具 -百度网盘

pan.baidu.com


2. 将 VScode_KR_workspace 文件夹放到 D 盘
可以放到其他地方，但要保证没有中文路径，也可以重命名，请改一个便于找到的位置与名字

  • 此文件夹包含配置文件，用于推荐插件与使用预先修改的设置，还包括 Love 2d 引擎

三、安装 Lua 扩展

  1. 打开 VSCode → 安装所有建议的扩展
    位于右下角
  • 包含 Lua 代码补全、诊断、格式化、用于调试的扩展
  • 所有插件均配置完毕，无需自行配置

四、打开游戏文件夹

  1. 将要修改的游戏文件全选复制到工作区文件夹内的 src 文件夹内
  2. 解压游戏本体到当前位置
  1. 通过 VSCode 打开文件夹

    注：以后修改的文件将会存在这个文件夹内
  • 设置与配置均已在 .vscode 内设置完毕，为了性能部分游戏资源文件已被排除（图片这些）

www.cnblogs.com

五、开启游戏的调试

  1. 反编译 src/version（以后 src 省略不写）模块
    反编译见 0.2 工具 > 反编译软件

  2. 将 build 键键值改为 "DEBUG"

六、运行

  1. 运行游戏
    注意：运行需要打开 steam
    点击运行与调试 → 选择模式：Debug → 运行
    Debug 为调试模式，会对性能造成较大影响
    Release 则为正常模式

  2. 打一些断点
    尽量打到会立刻执行的代码上，比如 kr/game_templates 内

  3. 一切正常下游戏会在断点处暂停，同时可以看到变量、堆栈

完成后注意一下存档位置会在 C:\Users\你的用户名称\AppData\Roaming\LOVE\kingdom_rush_xxx 里面

  • 也可以点击工作区文件夹内的存档位置快捷方式进行跳转

七、排查问题

  1. 如果运行后卡住没反应：检查 conf 模块关闭游戏自带控制台
    将 console 键键值从 true改为 false

  2. 如果在错误地方寻找 lua 文件可将 main 模块的 ppref 设为空字符串

可选

一、手动断点

在 args 模块的 love.keypressed 函数内打个断点即可

二、监视变量

注意：只有触发断点时监视才可用

变量
描述
game.game_gui.selected_entity 被鼠标点击选择的单位的表会显示在内
game.store 临时的表等（创建的实体等都在内）
game.game_gui 游戏 GUI（进入关卡后的技能、金币、生命等）
screen_map 地图的 UI 与 GUI 等（英雄殿堂、升级等）

断点

在运行过程中的打断点、删除、禁用断点、激活断点操作，只有断点后进行的才会立刻生效
