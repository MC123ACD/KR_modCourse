## 2.1 概念
### 2.1.1 ECS 架构
ECS 是 **Entity-Component-System**（实体-组件-系统）的缩写，是一种**游戏编程架构模式**。它的核心思想是：**把数据（是什么）和逻辑（做什么）分开**，通过“组合”而非“继承”来构建游戏对象。

1. **实体 - 容器**：
	* 实体本身**没有任何数据和行为**，它只是一个**唯一的 ID**，用来把不同的组件“打包”在一起。
	* 实体就像一个空的购物篮，你可以往里面放各种“组件商品”

2. **组件 - 纯数据**：
	* 组件只**存储数据**（属性），不包含任何逻辑。
	* 每个组件负责一个**单一的能力**（如血量、位置、速度等）。

3. **系统 - 纯逻辑**：
	- 系统只包含**行为逻辑**，不存储数据。
	- 每个系统**只关心拥有特定组件组合的实体**，并更新这些组件里的数据。

**示例**：创建并运作一个敌人
1. **创建实体**：生成一个唯一的 ID（比如 `entity #1`）。
2. **添加组件**：给这个实体附上：
   - `HealthComponent = { max_hp = 100, hp = 100, damage_taken = 50 }`
   - `SpriteComponent = { image = "enemy.png", pos = {x=400, y=500} }`
   - `MotionComponent = { speed = 50 }`
3. **系统运作**：
   - `HealthSystem` 找到 `entity #1`，读取 `damage_taken`，计算出新血量 `hp = 100 - 50 = 50`，写回组件。
   - `MovementSystem` 找到 `entity #1`，根据速度和帧时间更新 `SpriteComponent.pos`。
   - `RenderSystem` 找到 `entity #1`，读取 `image` 和 `pos`，在屏幕上画出敌人。

### 2.1.2 模板 - 批量生产实体的“蓝图”

如果一个游戏里有几十个相同的怪物，我们不可能每次都手动创建实体和添加组件。**模板**就是用来**预定义**一种实体的初始状态，然后通过**复制**快速生成多个实例。

**本质**：模板就是一个**完整的实体示例**（包含所有组件及其默认值）。创建新实体时，我们深度复制这个模板，得到独立的新实体。

**示例**：敌人模板

```lua
-- 定义模板库
templates = {}

-- 定义一个“普通敌人”模板
templates.enemy = {
    health = {
        hp_max = 100,
        hp = 100,
        damage = 50
    },
    sprite = {
        image = "enemy.png",
        pos = { x = 400, y = 500 }
    },
    motion = {
        speed = 50
    }
}

-- 辅助函数：根据模板名称创建实体
function create_entity(template_name)
    local template = templates[template_name]
    
    return table.deepclone(template)   -- 深拷贝，确保每个实体独立
end

-- 快速生成三个敌人
local enemy1 = create_entity("enemy")
local enemy2 = create_entity("enemy")
local enemy3 = create_entity("enemy")

-- 修改某个敌人的血量，不会影响其他敌人
enemy1.health.hp = 80
print(enemy2.health.hp)   -- 仍然是 100
```

#### 示例
```lua
-- 模板
templates = {}
-- 敌人模板
templates.enemy = {
	-- 血量组件
	health = {
		-- 最大血量
		hp_max = 100,
		-- 当前血量
		hp = 100,
		-- 受到的伤害
		damage = 50
	},
	-- 精灵组件
	sprite = {
		-- 显示的图像
		image = "enemy.png",
		-- 绘制位置
		pos = v(400, 500)
	},
	-- 移动组件
	motion = {
		-- 移动速度
		speed = 50
	}
}

-- 辅助函数，方便创建实体
function create_entity(template_name)
	local template = templates[template_name]
	-- 创建实体，深复制
	return table.deepclone(template)
end

-- 快速创建实体
-- 创建第一个敌人
enemy_1 = create_entity("enemy")
-- 创建第二个敌人
enemy_2 = create_entity("enemy")
-- 创建第三个敌人
enemy_3 = create_entity("enemy")
```

### 2.1.3 组件模板 - 进一步提高复用性

在上面的敌人模板中，我们手动写了 `health`、`sprite`、`motion` 三个子表。如果有很多种敌人，每个都重写这些组件会很繁琐。我们可以**先定义好各种“组件模板”**，然后在实体模板中直接引用并覆盖部分属性。

**示例**：组件模板 + 实体模板组合
```lua
-- 组件模板库
components = {
    health = {
        hp_max = 100,
        hp = 100,
        damage = 50
    },
    sprite = {
        image = "enemy.png",
        pos = { x = 0, y = 0 }
    },
    motion = {
        speed = 50
    }
}

-- 深拷贝辅助函数（同上）
function add_component(comp_name)
    local comp = components[comp_name]
    
    return table.deepclone(comp)
end

-- 实体模板库
templates = {}

-- 哥布林敌人
templates.goblin = {}
templates.goblin.health = add_component("health")
templates.goblin.sprite = add_component("sprite")
templates.goblin.sprite.image = "goblin.png"          -- 覆盖图像
templates.goblin.sprite.pos = { x = 100, y = 200 }    -- 覆盖初始位置
templates.goblin.motion = add_component("motion")
templates.goblin.motion.speed = 40                    -- 覆盖速度

-- 兽人敌人（高血量）
templates.orc = {}
templates.orc.health = add_component("health")
templates.orc.health.hp_max = 200
templates.orc.health.hp = 200
templates.orc.sprite = add_component("sprite")
templates.orc.sprite.image = "orc.png"
templates.orc.motion = add_component("motion")
templates.orc.motion.speed = 30

-- 创建哥布林和兽人
local goblin1 = create_entity("goblin")
local orc1 = create_entity("orc")
```

## 2.3 修改技能
### 2.3.1 近战攻击
拦截后的攻击，称为近战攻击（**melee**）。

```lua
tt.melee.attacks[1] = E:clone_c("melee_attack")	-- 返回深拷贝的组件
tt.melee.attacks[1].cooldown = 1				-- 冷却时间
tt.melee.attacks[1].hit_time = fts(8)			-- 攻击前摇
tt.melee.attacks[1].animation = "attack1"		-- 动画
tt.melee.attacks[1].sound = "MeleeSword"		-- 音效
tt.melee.attacks[1].xp_gain_factor = 2.5 		-- 获得的经验因子

tt.melee.attacks[2] = table.deepclone(tt.melee.attacks[1])	-- 复制攻击 1
tt.melee.attacks[2].chance = 0.5			-- 50% 概率进行攻击 2
tt.melee.attacks[2].animation = "attack2"	-- 表现为进行攻击 1，50% 播放 "attack2" 动画

tt.melee.range = 45	-- 拦截范围
```

### 2.3.2 修改伤害类型
通过 [[2. 实体与模板#2 1 3 组件模板]]，我们知道**组件其实是拥有默认值的**，后续可以通过**直接修改相应键值对来覆盖默认值**。

#### 查看组件的默认值
在 `components` 搜索 `melee` 组件
```lua
local melee = register_c("melee")
...
melee.attacks = {}
-- 镶套组件
melee.attacks[1] = CC("melee_attack")

-- 继续搜索 melee_attack 组件
local melee_attack = E:register_c("melee_attack")
...
melee_attack.damage_type = DAMAGE_PHYSICAL		-- 默认伤害类型：物伤

-- 模板内覆盖（修改）
tt.melee.attacks[1].damage_type = DAMAGE_TRUE	-- 修改为真伤
```
伤害类型同时也会决定击杀特效（炮伤爆浆，瓦解变成灰）

四个常用伤害类型：
- `DAMAGE_TRUE` 真伤
- `DAMAGE_PHYSICAL` 物伤
- `DAMAGE_MAGICAL` 法伤
- `DAMAGE_EXPLOSION` 炮伤

所有伤害类型见[[2. 常量#伤害类型]]

### 2.3.3 远程攻击
通过子弹（**bullet**）作为中介造成伤害的攻击，称为远程攻击（**ranged**）

由于是通过子弹造成伤害所以伤害与伤害类型等数据都是存储在子弹的模板上的，可以搜索子弹的模板名称来找到子弹的模板
```lua
-- 艾莉丹远程普攻
tt.ranged.attacks[1] = E:clone_c("bullet_attack")
tt.ranged.attacks[1].bullet = "arrow_hero_alleria"	-- 子弹
tt.ranged.attacks[1].bullet_start_offset = {
	v(0, 12)
}
tt.ranged.attacks[1].max_range = 150
tt.ranged.attacks[1].min_range = 45
tt.ranged.attacks[1].shoot_time = fts(6)
tt.ranged.attacks[1].cooldown = 0.6

-- 搜索 arrow_hero_alleria 的子弹模板
tt = RT("arrow_hero_alleria", "arrow")	-- arrow_hero_alleria 续承自 arrow 模板
-- 注：一些属性会在 hero.skills 中
tt.bullet.xp_gain_factor = 2.875
tt.bullet.prediction_error = false

-- 由于续承关系所以搜索 arrow 模板
-- all/templates 的 arrow 模板
local arrow = E:register_t("arrow", "bullet")
...
arrow.bullet.damage_type = DAMAGE_PHYSICAL	-- 默认伤害类型：物伤
```

### 2.3.4 技能
不属于以上两种攻击的攻击，称为技能（**timed_attacks**），例如：召唤实体、给予 buff 等

```lua
-- 熊人模板
tt.timed_attacks.list[1] = CC("spawn_attack")
tt.timed_attacks.list[1].animation = "ancestors"
tt.timed_attacks.list[1].cooldown = 14 + fts(40)

tt.timed_attacks.list[1].disabled = true
tt.timed_attacks.list[1].entity = "soldier_ingvar_ancestor"		-- 召唤这个实体

-- 维京先祖模板
tt = RT("soldier_ingvar_ancestor", "soldier_militia")		-- E:register_t
AC(tt, "reinforcement", "melee")		-- E:add_comps
...
tt.health.armor = 0.25
tt.health.hp_max = nil		-- 升级技能后动态赋值
tt.melee.attacks[1].damage_max = nil
tt.melee.attacks[1].damage_min = nil
tt.melee.attacks[1].cooldown = 1
tt.melee.attacks[1].hit_time = fts(15)
tt.melee.attacks[1].sound = "MeleeSword"
```

## 2.4 模板函数

### 2.4.1 E:register_t - 创建模板

```lua
E:register_t(模板名: str, 续承模板名?: str) -> 模板表的引用: table
```

**示例**:
```lua
local enemy = E:register_t("enemy")
enemy.health = 100
-- enemy = {"health" = 100}

local enemy_goblin = E:register_t("enemy_goblin", "enemy")
enemy_goblin.attack = 10
-- enemy_goblin = {"health" = 100, "attack" = 10}
```

### 2.4.2 E:add_comps - 增加组件

```lua
E:add_comps(模板: table, 组件名: str...)
```

#### 示例
```lua
component = E:add_comps("component")	-- 创建组件 “component”
component.num = 123
component.is_component = true

template = RT("template")
AC(template, "component")		-- 增加组件 “component”
template.n = 1
-- 此时 template = { component = { num = 123, is_component = true }, n = 1}
```

### 2.4.3 E:clone_c - 深拷贝组件
clone_c 全称 **clone componets** 简称 CC

```lua
E:clone_c(组件名: str) -> 深拷贝的表: table
```

#### 示例
```lua
component = RT("component")
component.num = 123
component.is_component = true

template = RT("template")
template.cc = CC("component")	-- 深拷贝组件，此时 cc 就是组件 component
template.num = 1
-- 此时 template = { cc = { num = 123, is_component = true }, num = 1 }
```

### 2.4.4 E:register_c - 创建组件
register_c 全称 **register component**

```lua
E:register_c(组件: str) -> 组件索引: table
```

#### 示例
```lua
a = E:register_c("a")	-- 创建一个名为 “a” 的组件
```

## 2.5 mod - 状态效果
造成的所有正面以及负面效果统称为状态效果（**modifiers**，缩写 mod）

### 2.5.1 攻击造成效果
1. 近战攻击直接在攻击后面增加 `mod` 键
2. 远程攻击增加到子弹模板的 `bullet` 键上，个别子弹无效（函数无实现）
```lua
-- 单位模板内
tt.melee.attacks[1].mod = 效果模板名: str
-- 子弹模板内
tt.bullet.mod = 效果模板名: str
```

#### 示例
```lua
-- 单位模板内
tt.melee.attacks[1].mod = "mod_lava"	-- 普通近战攻击造成燃烧效果

-- 子弹模板内
tt.bullet.mod = "mod_lava"	-- 子弹造成燃烧效果
```

**几个基础效果（位于 `template` 中）**
- `mod_stun` 眩晕
- `mod_slow` 减速
- `mod_freeze` 冻结
- `mod_lava` 燃烧
- `mod_poison` 中毒
- `mod_blood` 流血

### 2.5.2 修改效果
状态效果数据也是通过模板存储的，不能直接修改基础效果模板，由于续承关系直接修改会影响其他效果，所以我们需要创建一个新模板续承基础效果。

```lua
-- mod_lava（燃烧效果）模板
local mod_lava = RT("mod_lava", "modifier")
AC(mod_lava, "dps", "render")
mod_lava.modifier.duration = 2		-- 持续时间
mod_lava.dps.damage_min = 1			-- 最小伤害
mod_lava.dps.damage_max = 1			-- 最大伤害
mod_lava.dps.damage_inc = 3			-- 伤害系数
mod_lava.dps.damage_every = 0.2		-- 伤害频率
mod_lava.dps.damage_type = DAMAGE_TRUE	-- 伤害类型
...

-- 创建新模板 new_mod_lava 续承 mod_lava
tt = RT("new_mod_lava", "mod_lava")
tt.modifier.duration = 10		-- 持续时间 10 秒
tt.dps.damage_min = 5			-- 最小伤害 5
tt.dps.damage_max = 10			-- 最大伤害 10
-- 其他使用默认值

-- 单位模板内
tt.melee.attacks[1].mod = "new_mod_lava"	-- 使用新创建的效果

-- 子弹模板内
tt.bullet.mod = "new_mod_lava"	-- 使用新创建的效果
```

注：并非所有效果都要这样改，如果效果没有其他地方在用可以直接修改。

### 2.5.3 五代修改方法

```lua
-- balance 内
hero_spider = {
	basic_melee = {			-- 近战普攻
		mod = "mod_lava",	-- 增加 mod 键值对
		...
	}
}

-- 模板内
tt.melee.attacks[1].mod = b.basic_melee.mod	-- 读取 balance 内的表
```

### 2.5.4 效果互斥
效果互斥用于防止一些效果同时出现在同一个目标上。

```lua
tt.modifier.bans = {		-- 禁止的效果模板名
	效果模板名: str...
}
tt.modifier.ban_types = {	-- 禁止的效果类型，可选
	效果类型: str...
}
```

**几个效果类型**：
- `MOD_TYPE_BLEED` 流血
- `MOD_TYPE_POISON` 中毒
- `MOD_TYPE_FREEZE` 冻结
- `MOD_TYPE_STUN` 眩晕

所有效果类型见[[2. 常量#效果类型]]

#### 示例：
```lua
tt.modifier.bans = {		-- 禁止的效果模板名
	"mod_freeze",			-- 若目标上有燃烧效果，目标将不会被冻结
	...
}
tt.modifier.ban_types = {	-- 禁止的效果类型
	MOD_TYPE_FREEZE,		--  若目标上有燃烧效果，目标将不会被冻结类型的所有效果冻结
	...
}
```

### 2.5.5 移除禁止的效果
```lua
tt.modifier.remove_banned = 是否移除禁止的效果: bool
```

#### 示例
```lua
-- 燃烧效果模板内
tt.modifier.remove_banned = true
tt.modifier.bans = {
	"mod_freeze",		-- 如果目标上有冻结效果，移除其
	...
} 
tt.modifier.ban_types = {
	MOD_TYPE_FREEZE,	-- 如果目标上有冻结类型的效果，移除其
	...
}
```

### 2.5.6 效果叠加
三种模式（三选一）：
```lua
tt.modifier.allows_duplicates = 是否允许叠加: bool
tt.modifier.replaces_lower = 是否替换低级效果: bool
tt.modifier.resets_same = 是否仅重置持续时间: bool
```

#### 示例
```lua
-- 燃烧效果模板内
tt.modifier.allows_duplicates = true	-- 可叠加多个燃烧效果
-- 或
tt.modifier.replaces_lower = true	-- 替换低级燃烧效果
-- 或
tt.modifier.resets_same = true		-- 仅重置持续时间
```

## 2.6 特殊效果

### 2.6.1 范围攻击
给攻击初始化为范围攻击 `area_attack`，增加伤害范围即可。

```lua
tt.melee.attacks[1] = CC("area_attack")
tt.melee.attacks[1].damage_radius = 伤害范围: num
tt.melee.attacks[1].count = 最大目标数?: int
tt.melee.attacks[1].min_count = 释放限制，敌人数量?: int
```

#### 示例
```lua
tt.melee.attacks[1] = CC("area_attack")
tt.melee.attacks[1].damage_radius = 25	-- 伤害范围 25
tt.melee.attacks[1].count = 3			-- 最多对三名敌人造成伤害
tt.melee.attacks[1].min_count = 2		-- 需要两名敌人才能进行这个攻击
```

### 2.6.2 多次攻击
给攻击增加 `loops` 键，设定攻击次数即可。

```lua
-- 近战攻击
tt.melee.attacks[1].animations = {
	开始时的动画名: str|nil,
	连续攻击时的动画: str|nil,
	结束时的动画名: str|nil
}
tt.melee.attaks[1].loops = 连续攻击次数: int
tt.melee.attaks[1].hit_times = 一次攻击的次数（可指定间隔）: table{ 间隔: num, ... }
-- 可以理解成乘算，loops 为 0 不攻击： 攻击次数 = loops * hit_times

-- 远程攻击，同上 hit_times 变为 shoot_times
tt.ranged.attaks[1].shoot_times = 一次攻击的次数（可指定间隔）: table{ 间隔: num, ... }
```

#### 示例
```lua
-- 近战攻击
tt.melee.attacks[1].animations = {
	nil,		-- 开始动画
	"attack",	-- 连续攻击时的动画，复用普通近战攻击动画
	nil			-- 结束动画
}
tt.melee.attaks[1].loops = 2	-- 连续攻击 2 次
tt.melee.attaks[1].hit_times = { fts(10), fts(15) }	-- 一次攻击两次，第一次间隔 10 帧，第二次间隔 15 帧

-- 远程攻击
tt.ranged.attacks[1].animations = {
	nil,
	"shoot",	-- 复用普通远程攻击动画
	nil
}
tt.ranged.attaks[1].loops = 2
tt.ranged.attaks[1].shoot_times = { fts(10), fts(15) }	-- 一次攻击两次，第一次间隔 10 帧，第二次间隔 15 帧
```

### 2.6.3 子弹携带实体
给子弹的 `bullet.hit_payload` 设定实体模板名，即可让子弹落地后生成实体。

```lua
tt.bullet.hit_payload = 实体模板名: str
```

#### 示例
```lua
tt.bullet.hit_payload = "decal_bolin_mine"		-- 波林扔地雷
```

### 2.6.4 闪避
```lua
AC(tt, "dodge")		-- 增加闪避组件
tt.dodge.chance = 闪避概率: num
tt.dodge.silent = 是否有动画?: bool
```

#### 示例
```lua
AC(tt, "dodge")			-- 增加闪避组件
tt.dodge.chance = 0.25	-- 闪避概率 25 %
tt.dodge.silent = true	-- 无动画
```

### 2.6.5 标签
单位的 `vis` 组件的 `bans` 键可以让指定标签的攻击无视此单位（索敌过滤）

```lua
-- 单位模板
tt.vis.bans = 单位禁止的标签: int
tt.vis.flags = 单位的标签: int
tt.ranged.attacks[1].vis_bans = 攻击禁止的标签: int
tt.ranged.attacks[1].vis_flags = 攻击的标签: int
```

每个单位有自己的“身份证”（标签），攻击也有自己的“许可证”（标签）。双方互相检查，匹配不上就不能攻击。

**只看两个条件**：
1. **攻击的 `vis_bans`** 不能包含 **目标的标签**
2. **目标的 `vis_bans`** 不能包含 **攻击的标签**

#### 示例
```lua
-- 单位模板
tt.vis.bans = F_RANGED	-- 不会被远程攻击
tt.ranged.attacks[1].vis_bans = bor(F_BOSS, F_FLYING)	-- 远程攻击不攻击拥有 boss 标签与飞行标签的敌人
tt.ranged.attacks[1].vis_flags = F_RANGED				-- 远程攻击标签
```

#### 位运算增加多个标签
由于标签本质上是数字（通常为二次幂），可以通过 `bit.bor` 来增加多个标签。

```lua
bit.bor(十进制数字: int...) -> 合并后的十进制数字: int
```

**常用标签**：
- `F_BLOCK` 近战攻击
- `F_RANGED` 远程攻击
- `F_HERO` 英雄
- `F_BOSS` Boss
- `F_FLYING` 飞行
- `F_INSTAKILL` 秒杀

所有标签见[[2. 常量#标签]]

## 2.7 谁在加载模板与组件模块？
模块通常在 `E:load` 函数中加载

```lua
function entity_db:load()
	self.last_id = 1
	-- 初始化
	self.components = {}
	self.entities = {}
	
	-- 清除缓存，避免多次加载导致问题
	package.loaded.components = nil
	package.loaded.game_templates = nil
	package.loaded.templates = nil
	package.loaded.game_scripts = nil
	package.loaded.scripts = nil
	package.loaded.script_utils = nil
	
	-- 加载模板模块
	require("components")
	require("templates")
	require("game_templates")
end
```

## 2.9 本章总结

### 关键知识点回顾

1. **核心概念**：
	- **模板**：实体的数据蓝图，读取 `kr/game_templates` 和 `all/templates`
	- **组件**：为模板添加功能模块，读取 `all/components`
	- **实体**：使用模板创建的容器
	- `E.entities`：存放所有实体模板
	- `E.components`：存放所有组件
	- 数据流程：`balance` → 模板 → 实体 → 函数

2. **单位分类**：
	- **英雄**：具有 `hero` 组件
	- **士兵**：具有 `soldier` 组件的友方
	- **敌人**：具有 `enemy` 组件
	- **防御塔**：具有 `tower` 组件

3. **攻击类型**：
	- **近战攻击**：拦截后的攻击，使用 `melee` 组件
	- **远程攻击**：召唤子弹作为中介，使用 `ranged` 组件
	- **技能**：特殊攻击，使用 `timed_attacks` 组件

4. **效果**：
	- **造成效果**：给攻击或子弹的 `mod` 设定效果模板名
	- **修改效果**：增加一个新模板并续承原效果
	- **效果互斥**：给 `bans` 设定效果模板名
	- **移除被禁止效果**：`remove_banned` 设为真
	- **效果叠加**：`allows_duplicates` 允许叠加，`replaces_lower` 替换低级效果，`resets_same` 仅重置持续时间

5. **特殊效果**
	- **范围攻击**：使用 `area_attack` 组件，并设定范围
	- **多次攻击**：增加 `loops` 与 `hit_times`（对于近战攻击） 或 `shoot_times`（对于远程攻击）
	- **子弹携带实体**：给 `hit_payload` 设定实体模板名
	- **闪避**：增加 `dodge` 组件并增加概率
	- **标签**：使用 `bans` 与 `flags` 来过滤目标，使用 `bit.bor` 合并位运算增加多个标签

6. **加载组件与模板模块**：
	- 在 `E:load` 中初始化与加载
