1.1 模板

all/entity_db（以下简称 E） 返回的表内的 components 以及 entities 表内的表，称为模板

E 的基本内容：

E = {
	components = {}
	entities = {}
	
	-- 以及各种函数
}

如你所见 components 与 entities 表默认是空的，所以后面会通过 all/components 与 all/templates ，还有 kr/game_templates 模块将新模板增加进去（参考 E:load）

模块
作用
components 增加基础模板，用于创建模板时英雄、攻击等
templates 增加各种 mod 等模板
game_templates 增加一些英雄、防御塔、敌人、各种贴图和精灵等模板

game_templates 创建表时会读取 balance/balance 的各种值（五代特有）

创建模板

game_templates 等模块是怎样将新表增加进去的：

我们可以通过 E.entities.模板名 = xxx 来向表 entities 增加一些模板

也可使用等价的函数（实现方法同上）
E:register_t(模板名, 续承模板)（以下简称 RT） 来增加

续承模板可以理解为 E.entities.模板名 = deepclone(续承模板)（深拷贝）

Info

RT 会返回一个新增的表的索引

例如 tt = RT("hero_gerald", "hero") ，此时 tt 变量就是 E.entities.hero_gerald
注意重新赋值的影响

此外任何情况下都不建议用 E.entities.模板名，不想继承属性可以直接 RT(模板名)

向模板增加其他模板

我们查看 英雄 的基础模板发现只有一些等级相关的表，那么我们如何增加一些近战攻击，以及远程攻击的基础模板呢

正常人肯定可以想到 模板名.melee = deepclone(E.components[melee])

但是这种方法有些麻烦，我们可以使用官方提供的函数 E:add_comps(模板名, 基础模板...)（以下简称 AC）来向指定模板增加 n 个基础模板

Warning

注意：AC 只能增加 components 内的基础模板

索引模板

综上所述我们可以使用 E.entities.模板名 来直接索引模板

也可使用等价的函数 E:get_template(模板名)
