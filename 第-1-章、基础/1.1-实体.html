模板

使用模板（template）创建的英雄、士兵、敌人、防御塔、贴图、精灵等称为实体（entities）

创建流程见 1.4 函数 > 1 实体插入流程

在 all/entity_db（简称 E）返回的表中：

(1) entities 表中包含的子表，称为模板（template）
  • 核心功能：
    • 用于创建各种实体
    • 支持模板之间的深拷贝
(2) components 表中包含的子表，称为组件（components）
  • 核心功能：
    • 创建各种模板时会派生于组件
    • 为模板增加组件

组件与模板都是表，这里仅为了区分两个表

1. 模板存储结构

E = {
    components = { },  -- 组件（如血量、技能）
    entities = { },    -- 实体模板（如英雄、敌人）
    -- ... 其他函数
}
  • 初始状态：components 和 entities 为空，由其他模块动态填充

2. 模板来源

模块
用途
all/components 定义组件（如 health、melee）
all/templates 提供通用实体模板（如 Mod 相关）
kr/game_templates 定义游戏对象模板（英雄、防御塔等），会读取 balance/balance 数据

3. 模板操作 API

(1) 创建模板

  • 直接赋值 

    	E.entities.hero_gerald = { ... }  -- 手动创建（不推荐）

  • 推荐方法：E:register_t(模板名, 底表) （简称 RT）

    • 作用：模板会派生于底表（深拷贝）生成新模板

    • 返回值：新模板的引用（可直接操作）

    • 示例： 

    local tt = RT("hero_gerald", "hero")  -- tt 等价于 T("hero_gerald") 即 E.entities.hero_gerald
(2) 添加组件

  • 直接操作（不推荐）： 

    E.entities.hero_gerald.melee = table.deepclone(E.components.melee)

  • 推荐方法：E:add_comps(模板名, 组件， ...) （简称 AC）

    • 作用：向模板注入多个 components 中的组件

    • 限制：仅支持 components 中定义的组件

    • 示例： 

      AC("hero_gerald", "melee", "health")
(3) 索引模板

  • 直接索引（不推荐）： 

    local tpl = E.entities.hero_gerald

  • 推荐方法：E:get_template(模板名)（简称 T）

    • 注意：需确保模板已存在（通过 RT 或 AC 创建）
(4) 关键注意事项

  1. 避免直接操作 E.entities

    • 使用 RT / AC / T 可确保模板创建和组件注入的规范性

  2. 避免依赖模板的实现细节（照葫芦画瓢）

    • 通过在 components 搜索 AC / RT 的组件确认可用键，而非依赖具体模板的实现细节
      避免照葫芦画瓢，例如：官方在给 tt.melee.attacks[1]（近战攻击 1）增加属性时没有设置伤害类型（damage_type），并不代表不能修改伤害类型，通过在 components 搜索 AC 的 melee 组件，然后再搜索 CC 的 melee_attack 组件就可以找到伤害类型，所以我们可以用 tt.melee.attacks[1].damage_type = 伤害类型 来修改（默认为物伤 DAMAGE_PHYSICAL，所以官方才不设置伤害类型）

4. 组件行为

我们经常在修改模板的时候发现 melee，ranged， timed_attacks 组件，它们的用途如下

组件
 用途
近战攻击 melee 拦截后的攻击
近战普攻包含在内，还指具有此行为的技能
远程攻击 ranged 不进行拦截时的攻击
同上
技能 timed_attacks 召唤实体、特殊攻击、生成光环等
光环 aura 持续造成伤害、给予效果
效果 modifier 各种正面或负面效果（减速、流血、中毒等）

近战攻击与远程攻击我们称为基础攻击

拦截可以进行的远程攻击我们称为特殊远程攻击

5. 额外

(1) 技能类型组件

xxx_attack 组件都是技能类型组件

修改技能类型 API：
使用 E:clone_c(技能类型组件)（缩写 CC）函数，本质还是增加组件原理不赘叙

  • 示例：
    tt.melee.list[1] = CC("aura_attack")
范围攻击（aura_attack）：

只写关键的范围攻击，其他不再赘叙请自行查看

会将技能类型设置为 aura

技能将会变为范围攻击

注意：需要给技能的 damage_radius 键设置范围

(2) 多次攻击

只要给技能加上 loops 与 shoot_times 或 hit_times 键并设置次数，即可实现多次攻击

1. loops
  • 根据数字决定攻击次数
  • 例如：loops = 2
2. hit_times
  • 根据表的键数决定近战技能一次的攻击次数
  • 例如（一次攻击 2 次）：
{
	n,		-- n为攻击延迟
	fts(n)	-- n fts = n ÷ FPS，FPS 通常为 30
}
3. shoot_times
  • 根据表的键数决定远程技能一次的攻击次数
