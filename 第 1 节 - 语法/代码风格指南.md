
本篇风格指南列出了几条规定旨在让用户编写更为整洁、可读的代码，促进项目、讨论和教程之间的一致性。

> [!Info]+ 备注
> 
> 风格指南并不是硬性的规则教条，有些情况下，你可能无法施行下面的一些规范。如果这种情况发生在你身上，最好自行进行选择，并询问其他开发人员的见解。<br>
> 一般来说在项目和团队中保持代码风格的一致性，比一板一眼地遵循本指南更为重要。

## 1. 命名约定
1. 避免使用单个字母的变量名，除非作用域非常小（少于十行）。
2. 作用域较大的变量名应该更具描述性。
3. `i` 仅用作 `for` 循环（数字型 `for` 或 `ipairs`）中的计数器变量。
4. 使用 `pairs` 迭代时，除非编写操作泛型表的函数，否则优先使用更具描述性的名称而不是 `k` 和 `v`。
5. 使用 `_` 表示被忽略的变量（例如在 `for` 循环中）

| 类型  | 约定            | 规范示例               |
| --- | ------------- | ------------------ |
| 变量  | 蛇形命名法         | `player_health`    |
| 常量  | 全大写，单词之间使用下划线 | `MAX_PLAYERS`      |
| 函数  | 蛇形命名法         | `calculate_damage` |
| 类   | 帕斯卡命名法        | `TowerMenu`        |
| 文件名 | 蛇形命名法         | `game_file.lua`    |

> [!Success]+ 规范示例
> ```lua
> local damage = 100
> local health = 300
> 
> function take_damage()
> 	print("造成伤害" .. damage)
> 	health = health - damage
> end
> 
> for i = 1, 10 then
> 	print("循环体")
> end
> 
> for _, enemy in ipairs(enemies) do
> 	print(enemy)
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> local Number = 100
> local NUMBER = 300
> 
> function TakeDamage()
> 	print("造成伤害" .. Number)
> 	Number = NUMBER - Number
> end
> 
> for i, v in ipairs(enemies) do
> 	print(v)
> end
> ```

## 2. 缩进

使用 Tab 进行缩进，并保持统一。

> [!Success]+ 规范示例
> ```lua
> if condition then
>     print("缩进")
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> if condition then
>    print("三个空格")
>   print("两个空格")
>      print("五个空格")
> end
> ```

## 3. 空白行

用**空行分隔**逻辑上不同的代码段，以区分逻辑，增加可读性。

> [!Success]+ 规范示例
> ```lua
> local player_name = "小明"
> local player_level = 1
> 
> -- 角色状态
> local health = 100
> local damage = 50
> 
> -- 装备
> local weapon = "木剑"
> local armor = "布衣"
> ```

> [!Failure]+ 不规范示例
> ```lua
> local player_name = "小明"
> local player_level = 1
> -- 角色状态
> local health = 100
> local damage = 50
> -- 装备
> local weapon = "木剑"
> local armor = "布衣"
> ```

## 4. 表的格式

当表元素较多时，**每个元素应单独一行**，并加上适当的缩进。
并且**最后一行增加逗号**，这样在添加新元素时就不需要修改最后一行了。

> [!Success]+ 规范示例
> ```lua
> local towers = {"箭塔", "兵营", "法师塔"}
> local entities = {
> 	"箭塔",
> 	"兵营",
> 	"法师塔",
> 	"炮塔",
> 	"敌人1",
> 	"敌人2",
> 	"敌人3",
> }
> ```

> [!Failure]+ 不规范示例
> ```lua
> local entities = {"箭塔", "兵营", "法师塔", "炮塔", "敌人1", "敌人2", "敌人3"}
> 
> local entities = {
> 	"箭塔",
> 	"兵营",
> 	"法师塔",
> 	"炮塔",
> 	"敌人1",
> 	"敌人2",
> 	"敌人3"
> }
> ```

## 5. 表达式前后空格

降低代码紧凑性。

> [!Success] 规范示例
> ```lua
> local sum = 100 + 50
> local number = 123
> 
> function test(param1, param2)
> 	print("函数体")
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> local sum=100+50
> local number=123
> 
> function test(param1,param2)
> 	print("函数体")
> end
> ```

## 6. 一条语句一行代码

尽管在 Lua 中语句之间可以用**分号**分隔，但是应避免将多个语句放在一行当中，包括条件语句，使用换行来分隔语句。

> [!Success]+ 规范示例
> ```lua
> local a = 1
> local b = 2
> print(a + b)
> 
> if position.x > width then
> 	position.x = 0
> end
> if flag then
> 	print("flagged")
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> local a = 1; local b = 2; print(a + b)
> 
> if position.x > width then position.x = 0 end
> if flag then print("flagged") end
> ```

## 7. 对多行语句进行排版

将每行代码控制在 100 个字符以内。过长的行应适当拆分成多行提高可读性。拆分的语句应使用**两级缩进**。
将条件表达式拆分成多行时，`and` 和 `or` 关键字应当**放在下一行的开头**，而非上一行的结尾。

> [!Success]+ 规范示例
> ```lua
> -- 拆分参数列表
> local result = calculate_total_score(
>     player1_score,
>     player2_score,
 >    bonus_points
> )
> 
> -- 拆分条件
> if (
> 		position.x > 200
> 		and position.x < 400
> 		and position.y > 300
> 		and position.y < 400
> 	) then
> 	print("条件代码块")
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> local result = calculate_total_score(player1_score, player2_score, bonus_points)
> 
> if position.x > 200 and position.x < 400 and position.y > 300 and position.y < 400 then
> 	print("条件代码块")
> end
> ```

## 8. 避免冗余的括号

避免在表达式和条件语句中使用括号，除非需要修改操作顺序或者将语句拆分多行编写，否则只会降低可读性。

> [!Success]+ 规范示例
> ```lua
> if number == 100 then
> 	print("条件代码块")
> end
> ```

> [!Failure]+ 不规范示例
> ```lua
> if (number == 100) then
> 	print("条件代码块")
> end
> ```

## 9. 自动格式化

在 VSCode 中，可以按 `Shift + Alt + F` 自动格式化代码。它会帮你调整缩进、空格等，保持风格统一。
