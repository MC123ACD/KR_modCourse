### 9.1 LuaJIT 简介

LuaJIT 是一个**高性能的 Lua 解释器和即时编译器**（Just-In-Time Compiler）。它完全兼容 Lua 5.1 语法，但运行速度通常比标准 Lua 快很多——尤其是在循环和数值计算上。

## 9.2 使用 LuaJIT
切换到 LuaJIT 运行程序即可。

## 9.3 goto - 代码跳转

`goto` 可以在函数内部**直接跳转到指定的标签位置**。

**goto** 就像在地图上直接“传送”到某个标记点，跳过了中间的所有步骤。但滥用 `goto` 会让代码像一团乱麻，通常只在几种特定场景下使用。

```lua
::标签名::      -- 定义标签
goto 标签名     -- 跳转到该标签
```

#### 9.3.1 主要用途

##### 1. 模拟 continue 语句跳过本轮循环
```lua
for i = 1, 10 do
    if i % 2 == 0 then
        print("偶数: " .. i)
        goto continue		-- 直接跳过后面的打印
    end
    
    print("奇数: " .. i)
    
    ::continue::
end

--[[ 输出
奇数: 1
偶数: 2
奇数: 3
偶数: 4
...
]]
```

##### 2. 跳出多重嵌套
```lua
for i = 1, 3 do
    for j = 1, 3 do
        if i == 2 and j == 2 then
            print("找到目标，跳出所有循环")
            goto exit_loops
        end
        print(i, j)
    end
end

::exit_loops::
print("继续执行后续代码")
--[[ 输出
1	1
1	2
1	3
2	1
找到目标，跳出所有循环
继续执行后续代码
--]]
```

#####   3. 卫语句模式（提前返回）
```lua
-- 在非函数中模拟"提前返回"逻辑
local x = -1

::validate::
if x <= 0 then
    print("x 必须大于0")
    goto end_process
end

if x > 100 then
    print("x 不能超过100")
    goto end_process
end

print("x 有效: " .. x)

::end_process::
print("验证完成")
--[[ 输出
x 必须大于0
验证完成
]]
```

### 9.3.2 重要限制
- **不能跳进代码块**：不能从外部跳进代码块。
- **不能跳过局部变量的初始化**：如果跳过了 `local a = 10` 这一行，后面使用 `a` 可能会出错。

## 9.4 bit 库 - 位运算

```lua
local bit = require("bit")

local a = 0x0F 	-- 00001111
local b = 0xF0 	-- 11110000

bit.band(a, b)		-- 与
bit.bor(a, b)		-- 或
bit.bxor(a, b)		-- 异或
bit.bnot(a)			-- 非
bit.lshift(a, 2)	-- 左移
bit.rshift(b, 2)	-- 右移

--[[ 输出
1. bit库 - 位运算:
a = 0xF (15)
b = 0xF0 (240)
bit.band(a, b) = 0x0
bit.bor(a, b) = 0xFF
bit.bxor(a, b) = 0xFF
bit.bnot(a) = 0xFFFFFFFFFFFFFFF0
bit.lshift(a, 2) = 0x3C
bit.rshift(b, 2) = 0x3C
--]]
```

## 9.6 FFI

FFI 是 LuaJIT 最强大的特性之一，允许直接调用 C 函数、使用 C 数据类型和内存操作，无需编写 C 扩展模块。

**示例**：
```lua
local ffi = require("ffi")

-- 声明要用的 C 函数
ffi.cdef [[
    double sin(double x);
    double sqrt(double x);
]]

print("sin(90°) = " .. ffi.C.sin(math.pi / 2))  --> 1
print("sqrt(4) = " .. ffi.C.sqrt(4))    		--> 2
```

### 9.6.1 基本工作流程
1. 用 `ffi.cdef` 声明你想要调用的 C 函数和类型
2. 调用 `ffi.C.函数名` 来执行 C 函数
3. 用 `ffi.new`、`ffi.cast` 等创建和操作 C 数据

### 9.6.2 使用 C 结构体

```lua
local ffi = require("ffi")

ffi.cdef[[
    typedef struct {
        int x;
        int y;
    } Point;
]]

-- 创建 Point 实例（自动初始化为 0）
local p = ffi.new("Point")
p.x = 10
p.y = 20
print(p.x, p.y)   --> 10 20

-- 创建并初始化
local p2 = ffi.new("Point", {x = 30, y = 40})
print(p2.x, p2.y)   --> 30 40
```

### 9.6.3 内存管理 - 手动分配和释放

```lua
local ffi = require("ffi")

ffi.cdef [[
    void* malloc(size_t size);
    void free(void* ptr);
]]

-- 分配 10 个 int 的空间
local size = 10 * ffi.sizeof("int")
local buffer = ffi.C.malloc(size)

if buffer ~= ffi.NULL then
    -- 把这块内存当作 int 数组来用
    local int_array = ffi.cast("int*", buffer)

    for i = 0, 10 do
        int_array[i] = i * 2
        print(string.format("int_array[%d] = %d", i, int_array[i]))
    end

    -- 用完后一定要释放，否则内存泄漏
    ffi.C.free(buffer)
end

--[[ 输出
int_array[0] = 0
int_array[1] = 2
int_array[2] = 4
...
]]
```

> [!Warning]+ 警告
> FFI 不会自动帮你释放 C 分配的内存，你需要像 C 语言一样自己调用 `free`。忘记释放会导致内存泄漏。
