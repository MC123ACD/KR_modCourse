## 2.2 算术运算符

运算符是用来对数据进行操作的符号，算术运算符用于数学计算。

| 运算符 | 描述  | 示例       | 结果       |
| --- | --- | -------- | -------- |
| `+` | 加   | `10 + 3` | `13`     |
| `-` | 减   | `10 - 3` | `7`      |
| `*` | 乘   | `10 * 3` | `30`     |
| `/` | 除   | `10 / 3` | `3.3333` |
| `%` | 取余  | `10 % 3` | `1`      |
| `^` | 指数  | `10 ^ 3` | `1000`   |
| `-` | 负数  | `-10`    | `-10`    |

### 2.2.1 优先级

算术运算符的优先级同常规数学。

| 优先级 | 运算符         | 描述     |
| --- | ----------- | ------ |
| 1   | `()`        | 圆括号    |
| 2   | `^`         | 指数     |
| 3   | `-`         | 负号     |
| 4   | `*` `/` `%` | 乘、除、取余 |
| 5   | `+` `-`     | 加、减    |

**示例**：
```lua
print(2 + 4 * 4)	--> 18
print((2 + 4) * 4)	--> 24
print(-2 - 16 / 4)	--> -6
```

## 2.3 注释

注释是**对代码的解释或说明**，不会被 Lua 执行。良好的注释能让代码更容易理解和维护。

### 2.3.1 单行注释

只需要在**文本前方**增加 `--` 即可创建单行注释。

```lua
-- 单行注释：将 4 乘 4 的积与 2 相加
print(2 + 4 * 4)	-- 等于 18
```

### 2.3.2 多行注释

用 `--[[` 和 `]]` 包裹的注释可以跨越多行。

```lua
--[[
	这是一个多行注释
	可以写多行文字
	常用于文档说明或临时禁用一段代码
]]
print("这行代码会执行")

--[[
	临时禁用一段代码：
	print("这行代码不会执行")
]]
```

### 2.3.3 快速注释技巧

在 VSCode 中，选中代码后按 `Ctrl + /` 可以快速注释或取消注释。

```lua
-- print("被注释的代码")   -- 按 Ctrl + / 即可添加或删除注释
```

## 2.4 变量

**变量**就像一个带有名字的盒子，你可以往里面放东西（数据），也可以随时查看或更换里面的内容。它是程序存储、操作和传递数据的基本单元。

```lua
-- 声明一个变量，把字符串放进名为 text 的盒子里
local text = "Hello, Lua World!"

print(text) --> Hello, Lua World!
-- 可以重复使用这个盒子
print(text) --> Hello, Lua World!
print(text) --> Hello, Lua World!
```

### 2.4.1 变量的命名规则
1. 只能包含字母、数字和下划线（`_`），但不能以数字开头。
2. 区分大小写（`name` 和 `Name` 是两个不同的变量）。
3. 不能使用 Lua 的关键字（如 `if`、`while`、`local` 等）。

> [!Success]+ 合法示例
> ```lua
> player_name    -- 蛇形命名法（全小写，单词之间用下划线）
> playerName     -- 驼峰命名法（开头单词小写，单词首字母大写）
> PlayerName     -- 帕斯卡命名法（单词首字母大写）
> _player        -- 下划线开头
> player1        -- 包含数字（但不在开头）
> ```

> [!Failure]+ 非法示例
> ```lua
> 变量名          -- 错误：不能使用中文
> 1player        -- 错误：不能以数字开头
> player-name    -- 错误：不能包含连字符
> player name    -- 错误：不能包含空格
> local          -- 错误：不能使用关键字
> ```

> [!Info]+ 信息
> 命名惯例通常使用蛇形命名法

### 2.4.2 变量的类型

Lua 是**动态类型**语言，同一个变量可以存放任何类型的值，类型由当前赋的值决定。

```lua
local x = 10          -- 现在是数字
print(type(x))        --> number

x = "Hello"           -- 现在变成字符串
print(type(x))        --> string

x = true              -- 现在变成布尔值
print(type(x))        --> boolean

x = nil               -- 现在变成空
print(type(x))        --> nil
```

> [!Info]+ 信息
> `type` 用于检查变量的类型

### 2.4.3 变量的赋值与更新

你可以随时给变量赋新值，新的值会覆盖旧的。

```lua

local score = 0
print(score)          --> 0
score = 100           -- 更新为 100
print(score)          --> 100
score = score + 50    -- 先读旧值（100），加 50，再赋回给 score
print(score)          --> 150
```

### 2.4.4 多变量赋值

Lua 支持一次性给多个变量赋值，非常方便。

```lua
local a, b, c = 1, 2, 3

print(a, b, c)        --> 1 2 3

-- 也可以交换变量的值
local x, y = 10, 20
x, y = y, x

print(x, y)           --> 20 10
```

如果右边的值比左边少，多余的变量会被赋为 `nil`；如果右边值更多，多余的值会被忽略。

```lua
local p, q = 100
print(p, q)           --> 100 nil

local m, n = 5, 6, 7
print(m, n)           --> 5 6 （7 被丢弃）
```

### 2.4.6 `nil` 的作用

`nil` 表示“没有值”。当你声明一个变量但不赋值，它的值就是 `nil`。你也可以主动给变量赋 `nil` 来“清空”它。

```lua
local something
print(something)      --> nil

something = 42
print(something)      --> 42

something = nil       -- 清空
print(something)      --> nil
```

## 2.5 常见错误与调试

### 2.5.1 常见语法错误

| 错误类型      | 错误示例                         | 解决方法        |
| --------- | ---------------------------- | ----------- |
| 字符串未闭合    | `print("Hello)`              | 补全引号        |
| 函数名拼写错误   | `prin("Hello")`              | 检查拼写        |
| 使用中文标点    | `print（"Hello"）`             | 改用英文括号 `()` |
| 使用关键字作变量名 | `local end = 10`             | 更改变量名       |

### 2.5.2 简单调试技巧

1. **打印变量值**：用 `print` 输出变量内容。
```lua
local x = 10
print("x 的值是: " .. x)   -- 输出：x 的值是: 10
```

2. **临时禁用代码**：用注释 `--` 或多行注释 `--[[ ... ]]` 包裹暂时不需要的代码。

### 2.6.1 为什么要学会问 AI？

初学编程时，你会遇到大量问题。这些问题如果都询问他人等着解答，学习效率会很低。学会向 AI 提问，相当于有了一个**24 小时在线的私人老师**。

更重要的是，工作中没人会禁止你使用 AI。**会用 AI 解决问题本身就是一种核心能力**。

### 2.6.2 什么样的提问是好提问？

> [!Failure]+ 错误示范
> 问：Lua 报错了怎么办？
> AI 无法回答，因为信息太少。

> [!Success]+ 正确示范
> 问：我在写 Lua 程序，运行时报错 "`unexpected symbol near '1'`"，代码是：
> ```lua
> local 1player = 100
> print(1player)
> ```
> AI 会指出：**变量名不能以数字开头**。

### 2.6.3 提问模板

向 AI 提问时，包含以下 4 个要素：
1. **编程语言**：Lua
2. **目标**：想实现什么功能
3. **代码**：贴出具体代码
4. **遇到的问题**：报错信息/不符合预期的地方

### 2.6.4 示例

#### 示例 1：理解报错信息
问：Lua 中这段代码报错：
```lua
local score = 95
priny(score)
```
报错信息：`attempt to call global 'priny' (a nil value)`

AI回答：`priny` 写错了应该是 `print`

#### 示例 2：学习新知识
问：Lua 中除了加减乘除，还有其他数学运算吗？我想开平方根。

AI回答：使用 `math.sqrt` 开平方根：
```lua
print(math.sqrt(9))		--> 3
print(math.sqrt(25))	--> 5
```

### 2.6.5 注意事项

- **AI 也会犯错**：如果答案不对劲，可以追问或换种问法
- **先思考再问**：自己先尝试解决，解决不了再问 AI
- **理解而非复制**：看懂 AI 的解答，不要直接复制粘贴
- **可以追问**：第一次没听懂？继续问，直到明白为止
