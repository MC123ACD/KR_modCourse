
控制结构是编程语言的骨架，它决定了程序的执行流程。本章将系统地学习 Lua 中的条件判断、循环语句、代码块和变量作用域，帮助你编写出逻辑清晰、功能强大的程序。

## 2.1 条件语句

条件语句让程序有了“判断能力”——**根据条件是否成立，决定要不要执行某段代码**。

### 2.1.1 布尔值：什么是真，什么是假？

Lua 用**布尔值**来表示“是”与“否”：
- `true` —— 真，表示“成立”
- `false` —— 假，表示“不成立”

> [!Info]+ 信息
>在 Lua 中，只有 `false` 和 `nil` 被认为是“假”，其他**所有**值（包括数字 `0`、空字符串 `""`、空表 `{}` 等）都被视为“真”。

### 2.1.2 基础 if - 最简单的判断

**语法**：
```lua
if 条件 then
    -- 条件为 true 时执行的代码
end
```
- **条件**：一个表达式，Lua 会把它当作“真”或“假”来判断。
- 如果条件为真，就执行 `then` 到 `end` 之间的代码；否则直接跳过。

> [!Info]+ 信息
> - `if`、`then`、`end` 是 Lua 的关键字，要完整写出。
> - 条件里可以用算术、比较、逻辑运算，组合成复杂的表达式。
> - `then` 和 `end` 之间的代码块可以写多行，里面甚至可以再嵌套别的结构。

**示例**：
```lua
local health = 75

if health > 0 then
    print("玩家还活着")
end

--> 玩家还活着
```

### 2.1.3 if-else 语句 - 二选一

当你想在条件成立和不成立时执行**不同的代码**时，就用 `else`。

**语法**：
```lua
if 条件 then
    -- 条件为 true 时执行
else
    -- 条件为 false 时执行
end
```

**示例**：
```lua
-- 玩家当前血量
local health = 0

-- 血量大于 0 玩家未死亡
if health > 0 then
    print("玩家还活着，当前血量：" .. health)
-- 血量小于 0 玩家死亡
else
    print("玩家已死亡")
end

--> 玩家已死亡
```

> [!Info]+ 信息
> `else` 分支是可选的，且只能有一个。

### 2.1.4 if-elseif-else 语句 - 多选一

当你有多个条件需要按顺序判断时，可以使用 `elseif` 链。

**语法**：
```lua
if 条件1 then
    -- 条件 1 为 true 时执行
elseif 条件2 then
    -- 条件 2 为 true 时执行
elseif 条件3 then
    -- 条件 3 为 true 时执行
else
    -- 所有条件都为 false 时执行
end
```

**执行流程**：
1. 从上到下依次判断每个条件。
2. 遇到第一个为真的条件，就执行对应的代码，然后跳出整个 `if` 结构。
3. 如果所有条件都为假，就执行 `else` 对应的代码（如果有的话）。

**示例**：游戏难度选择
```lua
-- 难度：困难
local difficulty = "hard"
-- 敌人与玩家血量初始化
local enemy_health, player_health

-- 简单难度的敌人与玩家血量
if difficulty == "easy" then
    enemy_health = 50
    player_health = 200
-- 普通难度的敌人与玩家血量
elseif difficulty == "normal" then
    enemy_health = 100
    player_health = 150
-- 困难难度的敌人与玩家血量
elseif difficulty == "hard" then
    enemy_health = 150
    player_health = 100
-- 避免填写错误的难度导致的问题
else
    print("无效难度，使用普通难度")
    enemy_health = 100
    player_health = 150
end
```

> [!Info]+ 信息
> - `elseif` 是一个单词，中间没有空格。
> - `elseif` 可以有任意多个（包括零个）。

### 2.1.5 嵌套 if 语句 - 判断里的判断

`if` 语句内部还可以再放 `if` 语句，形成嵌套结构。这就像“如果外面条件满足，再看看里面的条件”。

**示例**：
```lua
-- 玩家等级
local level = 10
-- 模式
local mode = "daytime"

-- 限制玩家等级
if level >= 5 then
	print("进入冒险模式")

    if mode == "night" then
        print("进入夜晚模式")
    else
        print("进入白天模式")
    end
else
    print("等级不足，无法进入冒险模式")
end
```

> [!Warning]+ 警告
> 嵌套层数不宜过深，否则代码可读性会下降。

### 2.1.6 常见错误

**错误 1**：忘记写 `then`
```lua
if x > 10        -- 缺少 then
    print("x 大于 10")
end
```

**错误 2**：多分支条件重叠
```lua
local level = 11

if level > 10 then
    print("解锁技能：火球术")
elseif level > 20 then   -- 此条件永远不会进入，因为 > 10 已经成立
    print("解锁技能：治疗")
end
```

**正确做法**：将范围从大到小排列，或使用精确区间。
```lua
local level = 10

if level > 20 then
    print("解锁技能：治疗")
elseif level > 10 then
    print("解锁技能：火球术")
end
```

## 2.2 比较运算符

比较运算符就像生活中的“比大小”，用来判断两个值之间的关系，结果只有两种：**成立（true）** 或 **不成立（false）**。

| 运算符  | 描述   | 示例（假设 `x = 10`）     |
| ---- | ---- | ------------------- |
| `==` | 等于   | `x == 10` → `true`  |
| `~=` | 不等于  | `x ~= 5` → `true`   |
| `<`  | 小于   | `x < 20` → `true`   |
| `>`  | 大于   | `x > 10` → `false`  |
| `<=` | 小于等于 | `x <= 10` → `true`  |
| `>=` | 大于等于 | `x >= 15` → `false` |

**示例**：
```lua
local x, y = 10, 20
print(x == y)   --> false   -- 10 等于 20？不成立
print(x ~= y)   --> true    -- 10 不等于 20？成立
```

> [!Warning]+ 警告
> `==` 是比较，`=` 是赋值。写条件时一定用 `==`，否则程序会报错。

## 2.3 逻辑运算符

逻辑运算符用来**组合多个条件**，让判断更灵活。Lua 支持三个逻辑运算符：`and`、`or` 和 `not`。

### 2.3.1 and - 两个都要真

- 如果第一个操作数为“假”（`false` 或 `nil`），返回第一个操作数；
- 否则返回第二个操作数。

```lua
print(true and false)    --> false   -- 一个真一个假 → 假
print(false and true)    --> false   -- 同上
print(true and "hello")  --> "hello" -- 真 → 返回第二个值
print(nil and 100)       --> nil     -- 假 → 返回第一个值
```

**实际用途**：检查多个条件**同时成立**。
```lua
local level = 25
local can_get_skill = true

if level >= 25 and can_get_skill then
    print("可以学习技能")		-- 等级达标并且有资格
else
	print("不能学习技能")		-- 等级不达标或没有资格
end

--> 可以学习技能
```

### 2.3.2 or - 至少一个真

- 如果第一个操作数为“真”（非 `false` 和 `nil`），返回第一个操作数；
- 否则返回第二个操作数。

```lua
print(true or false)     --> true     -- 第一个为真，直接返回
print(false or true)     --> true     -- 第一个为假，返回第二个
print(nil or "default")  --> "default"-- nil 为假，返回默认值
print(100 or 200)        --> 100      -- 100 为真，直接返回
```

**实际用途**：检查多个条件**至少一个成立**。
```lua
local is_weekend = false
local is_holiday = true

if is_weekend or is_holiday then
    print("今天是休息日")   -- 是周末或节假日
else
	print("今天是工作日")   -- 不是周末并且不是节假日
end

--> 今天是休息日
```

### 2.3.3 not - 取反

- 对真值取反得 `false`，对假值取反得 `true`。
- **注意**：`not` 总是返回布尔值 `true` 或 `false`，不会返回其他值。

```lua
print(not true)    --> false
print(not false)   --> true
print(not nil)     --> true     -- nil 是假，取反得真
print(not 0)       --> false    -- 0 在 Lua 里是真，取反得假
```

**实际用途**：判断“不满足某个条件”。
```lua
local is_death = false

if not is_death then
    print("玩家没有死亡")		-- is_death 为假，取反后为真
else
	-- else 可以理解为取反，相对于 not not is_death 条件，两次取反，即为真
	print("已玩家死亡")		-- is_death 为真时条件成立
end
```

### 2.3.4 实用技巧

#### 1. 用 `or` 给变量设置默认值

当你希望一个变量有“保底值”时，`or` 非常方便：

```lua
local health = 100
local damage   -- 没有赋值，默认为 nil

-- 假设受到攻击
damage = 10

-- 如果 damage 是 nil（没有受伤），就默认扣 1 点血
damage = damage or 1

health = health - damage
print("剩余血量：" .. health)  -- 有伤害时扣 10，没伤害时扣 1
```

> [!Info]+ 信息
> 这是 Lua 中非常经典的写法：`变量 = 变量 or 默认值`

#### 2. 用 `and` 和 `or` 模拟三元运算符

其他语言里有 `条件 ? 值1 : 值2` 这样的三目运算符，Lua 没有，但可以用 `and` 和 `or` 组合实现：

```lua
local is_enemy = true

local message = is_enemy and "攻击敌人" or "无法攻击"
print(message)   --> 攻击敌人
```

**原理**：
- 如果 `is_enemy` 为真，`and` 返回第二个值（"攻击敌人"），`or` 看到第一个值为真，直接返回，最终得到 `"攻击敌人"`。
- 如果 `is_enemy` 为假，`and` 返回第一个值（`false`），`or` 看到第一个值为假，返回第二个值（"无法攻击"）。

> [!Warning]+ 警告
> 这种写法仅在第二个表达式（`and` 后面的值）不会为 `false` 或 `nil` 时安全。如果那个值本身就是假，结果会出乎意料。初学者可以先了解，熟练后再使用。

## 2.4 代码块与作用域

### 2.4.1 什么是代码块？

代码块就是一组被“包起来”的语句。它像一个个**房间**，房间里的东西（变量）只在房间内可见。

常见的代码块有：
- `do ... end` —— 手动创建的“临时房间”
- `if 条件 then ... end` —— 条件判断的房间
- `while ... do ... end`、`for ... do ... end` —— 循环的房间
- `function ... end` —— 函数的房间
- 整个文件本身也是一个代码块，叫**主块**。

### 2.4.2 局部变量 - 自己的房间

用 `local` 声明的变量只在它所在的代码块内有效。

```lua
local outer = "我在外面"

do   -- 开一个新房间
    local inner = "我在里面"
    print(outer)  -- 里面的房间能看到外面的 → 输出：我在外面
end

print(inner)  -- 外面的房间看不到里面 → 输出：nil
```

**关键规则**：
- ✅ 内部可以访问外部
- ❌ 外部不能访问内部

#### 变量的遮蔽 - 同名变量的“近水楼台”

当内外有同名变量时，内部的变量会“遮住”外部的，就像近处的东西挡住了远处的。

```lua
local state = "我在外面"

do
    local state = "我在里面"   -- 和外部的同名
    print(state)   --> 我在里面   （内部优先）
end

print(state)   --> 我在外面   （出来后又恢复）
```

#### 最佳实践

1. **尽量使用 `local` 声明**  
   - 可以避免命名冲突，提升性能。

2. **一次声明多个变量**  
```lua
local x, y, z = 10, 20, 30
```

3. **先声明，后赋值（尤其是复杂逻辑）**  
```lua
local player_name
local player_score

-- ... 一些处理
player_name = "小明"
player_score = 1000
```

4. **利用 Lua 的多重赋值交换变量**  
```lua
local a = 10
local b = 10

a, b = b, a   -- 交换后 a = 20, b = 10
```

### 2.4.3 全局变量 - 公共广场

不用 `local` 直接赋值的变量就是**全局变量**，它在整个程序里都可见，像是公共广场上的公告板。

```lua
global_counter = 0   -- 创建全局变量

do
    global_counter = global_counter + 1   -- 任何地方都能改
end

print(global_counter)  --> 1
```

**特点**：
- 无需声明，直接赋值即创建。
- 任何地方都能访问和修改，很容易“打架”（命名冲突）。
- 访问不存在的全局变量返回 `nil`，不会报错。

> [!Warning]+ 警告
> **常见陷阱**：忘记写 `local`，意外创建了全局变量
> ```lua
> do
>     a = 100   -- 忘了 local，变成了全局变量
> end
> 
> print(a)  --> 100 （本意可能只是想临时用一下）
> ```

## 2.5 循环语句

循环让程序可以重复执行某段代码，直到条件不满足为止。

### 2.5.1 为什么需要循环？

如果让你打印 1 到 100，没有循环就得写 100 行 `print(1)`、`print(2)` ……有了循环，几行代码就搞定。

### 2.5.2 for 循环（数值型）

**适用场景**：遍历固定范围、计数、数组遍历等 **“次数已知”** 的情况。

**语法**：
```lua
for 循环变量 = 起始值, 结束值, 步长 do
    -- 循环体
end
```
- 步长默认为 1（可省略）

**示例**：
```lua
-- 正向计数，步长默认为 1
for i = 1, 5 do
    print(i)   --> 1 2 3 4 5
end

-- 倒着数，步长 -1
for i = 5, 1, -1 do
    print(i)   --> 5 4 3 2 1
end
```

> [!Info]+ 信息
> - 循环变量自动递增（或递减），无需手动维护。
> - 循环次数在开始时就已经确定。
> - 循环变量只在循环内有效。

> [!Warning]+ 警告
> 循环体使用 `do-end` 包裹而非 `then-end`

### 2.5.3 中断循环：break

`break` 用于立即退出当前循环。

```lua
for i = 1, 10 do
	print(i)
	
    if i == 5 then
        print("到达 5，退出循环")
        break
    end
end
--[[ 输出
1
2
3
4
5
到达 5，退出循环
--]]
```

> [!Info]+ 信息
> - `break` 只能在循环内使用，不能单独用在 `if` 里（除非 `if` 在循环内）。
> - 多层循环中，`break` 只退出**最内层**的循环。

### 2.5.4 跳过本轮 - continue

Lua **没有** `continue` 语句（跳过本轮剩余代码，进入下一轮），但可以用 `if` 实现相同效果。

```lua
for i = 1, 10 do
	-- 跳过偶数（相当于 continue）
    if i % 2 ~= 0 then
    	print("奇数：" .. i)
    end
end
--[[ 输出
奇数：1
奇数：3
奇数：5
奇数：7
奇数：9
--]]
```

### 2.5.5 while 循环 -  先检查，后执行

**适用场景**：等待某个条件成立、处理不确定长度的数据、游戏主循环等 **“次数未知，条件驱动”** 的情况。

**语法**：
```lua
while 条件 do
    -- 循环体（条件为真时执行）
end
```

**执行逻辑**：先检查条件，如果条件为真，就执行循环体；执行完后再检查条件，如此反复，直到条件为假时退出。

**示例**：猜数字游戏（直到猜对为止）
```lua
local secret = math.random(1, 100)   -- 随机生成 1-100 的秘密数字
local guess = 0
local attempts = 0

print("猜猜我心里想的数字（1-100）")

while guess ~= secret do
    guess = 50   -- 这里假设玩家每次都猜 50（演示用）
    attempts = attempts + 1
    
    if guess < secret then
        print("猜小了")
    elseif guess > secret then
        print("猜大了")
    else
        print("恭喜！猜对了！")
    end
end

print("你用了 " .. attempts .. " 次猜中")
```

> [!Warning]+ 警告
> 千万别忘记改变条件，除非是有意为之。
> 
> ```lua
> local i = 1
> while i <= 5 do
>     print(i)   -- 没有 i = i + 1
> end            -- 死循环！程序会一直打印 1
> ```

### 2.5.6 repeat-until 循环 - 先执行，后检查

**语法**
```lua
repeat
    -- 循环体（至少执行一次）
until 条件   -- 条件为真时停止循环
```

**执行逻辑**：先执行一次循环体，然后检查条件。如果条件为假，就继续重复执行；如果条件为真，就退出循环。

**示例**：输入密码
```lua
local password

repeat
    print("请输入密码（至少6位）：")
    
    -- 模拟用户输入（这里用变量代替，实际中可能是 io.read()）
    password = "123"   -- 假设用户第一次输入太短
    
    if #password < 6 then
        print("密码太短，请重新输入")
    end
until #password >= 6   -- 直到密码长度满足要求才退出

print("密码设置成功！")
--[[ 输出：
请输入密码（至少6位）：
密码太短，请重新输入
请输入密码（至少6位）：
（第二次输入正确后）密码设置成功！
]]
```

### 2.5.7 while 与 repeat-until 对比

| 循环类型           | 检查时机 | 最少执行次数 | 条件的含义     |
| -------------- | ---- | ------ | --------- |
| `while`        | 开头   | 0 次    | 为真时**继续** |
| `repeat-until` | 结尾   | 1 次    | 为真时**结束** |

```lua
-- while 可能一次都不执行
local i = 10

while i < 5 do
    print(i)   -- 不输出
end

-- repeat-until 至少执行一次
i = 10

repeat
    print(i)   -- 输出 10（一次）
until i < 5
```

## 2.6 VSCode 使用技巧

### 2.6.1 折叠代码
将鼠标悬停在行号左侧的三角上，可折叠代码块（如 `if`、`for` 等）。

### 2.6.2 自定义折叠区域
使用 `--#region` 和 `--#endregion` 标记可以折叠任意代码段。

```lua
--#region 变量
local a = 1
local b = 2
local c = 2
--#endregion
```

大纲可以快速查看代码结构，表结构等。

- `Alt + 鼠标左键`：添加多个光标。

2. 在不改变现有 `do ... end` 结构的前提下，修改以下代码，使得最内层的 `x`（值为 30）能在最外层被访问到。

