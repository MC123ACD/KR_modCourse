## 4.1 什么是表？

表（table）是 Lua 中**唯一**的数据结构，但它极其强大和灵活。你可以把它想象成一个**万能的抽屉柜**——每个抽屉都有编号（数字索引）或标签（键名），你可以往里面放任何东西，甚至还能放其他抽屉柜。

通过表，Lua 可以实现数组、字典、对象、模块等几乎所有复杂数据结构。

## 4.2 表的创建和初始化

### 4.2.1 创建空表

```lua
local empty_table = {}
```

### 4.2.2 数组初始化（带编号的抽屉）

```lua
local monsters = {"僵尸", "骷髅", "蜘蛛", "土匪"}
```

> [!Info]+ 信息
> 这种写法会自动给元素分配连续的整数索引，从 **1** 开始：
> - `monsters[1]` → "僵尸"
> - `monsters[2]` → "骷髅"
> - `monsters[3]` → "蜘蛛"
> - `monsters[4]` → "土匪"

#### 创建不连续的数组（手动指定编号）

```lua
local monsters = {
	-- 索引 1 对应 "僵尸"
	[1] = "僵尸",
	-- 索引 3 对应 "僵尸"
	[3] = "蜘蛛", 
	-- 索引 100 对应 "史莱姆王"
	[100] = "史莱姆王",
}
```

### 4.2.3 字典初始化（带标签的抽屉）

```lua
local player = {
    name = "李雷",
    age = 18,
    class = "战士",
    level = 5,
}

-- 也可以写成方括号形式（效果一样，但通常点号更简洁）
local player = {
    ["name"] = "李雷",
    ["age"] = 18,
    ["class"] = "战士",
    ["level"] = 5,
}
```

#### 复杂键字典的创建

当键名包含空格、连字符等特殊字符，或者本身就是布尔值、数字时，必须用方括号 `[]` 包裹。

```lua
local complex_keys = {
    ["full name"] = "张三",	-- 包含空格
    ["score-100"] = 95,		-- 包含连字符
    [true] = "布尔键",		-- 布尔值作为键
    [123] = "数字键",		-- 数字作为键（数组也属于这类）
}
```

### 4.2.4 混合初始化（数组部分+字典部分）

```lua
local mixed = {
    "第一个元素",		-- 自动分配索引 1
    "第二个元素",		-- 自动分配索引 2
    author = "小明",	-- 字典部分，键 "author"
    version = 1.0	-- 字典部分，键 "version"
}
```

### 4.2.5 嵌套初始化（抽屉里放抽屉）
```lua
local game_world = {
    name = "魔法世界",
    players = {
        {name = "战士", level = 10},
        {name = "法师", level = 8}
    },
    settings = {
        difficulty = "normal",
        sound = true,
        graphics = "high"
    }
}
```

## 4.3 表的操作

### 4.3.1 访问表中元素

你可以用点号 `.` 或方括号 `[]` 来取出抽屉里的东西。

```lua
local player = {
    name = "小明",
    level = 10,
}

print("玩家姓名: " .. player.name)		--> 玩家姓名: 小明
print("玩家等级: " .. player.level)		--> 玩家等级: 10
print("玩家姓名: " .. player["name"])	--> 玩家姓名: 小明
print("玩家等级: " .. player["level"])	--> 玩家等级: 10
print("其他: " .. player["else"])		--> nil（不存在的键返回 nil）
```

> [!Info]+ 信息
>- `player.name` 和 `player["name"]` 等价，但**非法命名的键**（如包含空格、连字符）只能用方括号。
> - 访问不存在的键会得到 `nil`，不会报错。

#### 镶套表的访问

```lua
local game_world = {
    world_name = "魔法世界",
    players = {
        {name = "战士", level = 10},
        {name = "法师", level = 8}
    }
}

print("玩家姓名: " .. game_world.players[1].name)        --> 玩家姓名: 战士
print("玩家等级: " .. game_world.players[1].level)       --> 玩家等级: 10
```

#### 安全访问表的技巧：避免“nil 不能点”的错误

在 Lua 中，如果你直接写 `game_data.players[1].name`，万一 `game_data.players` 是 `nil`，程序就会报错，因为 `nil` 不能使用 `.` 或 `[]` 访问元素。

**为什么出错？**  
就像你有一个盒子（表），你想拿盒子里的小盒子（`players`）里的东西（`[1].name`）。如果中间某个盒子根本不存在，你就没法继续往里拿，程序就崩溃了。

**解决方案：**  
用 `and` 和 `or` 组合，**一步一步地检查**，只要中间有一步是 `nil`，整个表达式就停下来，不会报错。

```lua
local game_data = {}  -- 空表，没有 players 这个键

-- 危险写法：直接访问会报错（nil 不能点）
-- local name = game_data.players[1].name

-- 安全写法：用 and 短路求值，逐个检查
local player_name = game_data.players and game_data.players[1] and game_data.players[1].name

-- 如果任何一个环节是 nil，player_name 就是 nil
print("第一个玩家名称: " .. (player_name or "未知"))  -- 输出：未知
```

> [!Info]+ 信息
> **原理**：`and` 运算中，左边为假（`nil` 或 `false`）就直接返回左边，不再计算右边，从而避免了访问 `nil` 的属性。

### 4.3.2 直接修改与增加元素

修改表中的元素，就像打开抽屉换掉里面的东西。

#### 1. 数组部分
```lua
local monsters = {"僵尸", "骷髅", "蜘蛛", "土匪"}
monsters[1] = "恶魔"   -- 把第一个抽屉里的僵尸换成恶魔

print("第一个怪物: " .. monsters[1])	--> 第一个怪物: 恶魔
```

> [!Warning]+ 警告
> 不能用 `monsters.1` 访问数组元素，数字索引只能用方括号。

#### 2. 字典部分
```lua
local inventory = {}

-- 添加元素到字典部分
inventory.gold = 100
inventory.silver = 50
inventory.copper = 200
-- 等价写法（不推荐，除非键名特殊）
inventory["gold"] = 100
inventory["silver"] = 50
inventory["copper"] = 200

print("gold: " .. inventory.gold)		--> gold: 100
print("silver: " .. inventory.silver)	--> silver: 50
print("copper: " .. inventory.copper)	--> copper: 200
```

### 4.3.3 表的长度运算符

`#` 返回表**数组部分**最后一个连续整数索引。它只对“从 1 开始连续”的数组部分准确，遇到空洞（`nil`）就会停止计数。

```lua
-- 连续数组
local arr1 = {1, 2, 3, 4, 5}
print(#arr1)   --> 5

-- 有空洞的数组
local arr2 = {1, nil, 3, 4, 5}
print(#arr2)   --> 1（遇到第一个 nil 就停止）

-- 不连续但中间有空洞
local arr3 = {1, 2, 3, 5}
print(#arr3)   --> 3（只计算了 1,2,3）

-- 字典部分不影响长度
local mixed = {"a", "b", "c", name = "测试"}
print(#mixed)  --> 3（只算数组部分）
```

## 4.4 值类型与引用类型

### 4.4.1 值类型

数字、字符串、布尔、`nil` 是值类型，当其被赋值给另一个变量时会自动创建**副本**，修改变量时原值不会被影响。

```lua
local num1 = 123
local num2 = num1	-- 自动创建 num1 副本

num2 = 666

print(num1)		--> 123（num1 没有被 num2 的赋值影响）
print(num2)		--> 666
```

### 4.4.2 引用类型

表是**引用类型**。当你把一个表赋值给另一个变量时，它们指向的是**同一个表**，而不是复制一份。

```lua
local original = {x = 10, y = 20}
local reference = original  -- 不是复制，是引用同一个表

reference.x = 100  -- 修改引用也会影响原始表
print(original.x)  --> 100

-- 如果需要复制表，需要手动操作（后面会讲）
```

> [!Info]+ 信息
> **为什么这样设计？** 因为表可能很大，如果每次赋值都复制一份，会浪费大量内存和时间。通过传递引用（类似指针），多个变量可以高效地共享同一个数据。

#### 比较引用类型

引用类型比较的是引用，而不是内容。

```lua
local same1 = {1, 2, 3}
local same2 = same1
local different = {1, 2, 3}

-- 打印引用类型输出的是引用
print(same1)		--> table: 66C0
print(same2)		--> table: 66C0
print(different)	--> table: 0B88

print(same1 == different)	--> false（不同表，尽管内容相同）
print(same1 == same2)		--> true（同一表）
```

## 4.5 表遍历

### 4.5.1 ipairs - 遍历数组部分

`ipairs` 专门用于遍历数组部分（索引从 1 开始连续），遇到 `nil` 就停止。

**语法**：
```lua
for 索引, 值 in ipairs(表) do
	-- 循环体
end
```
- `索引` 会自动从 1 开始递增，直到遇到第一个 `nil`。
- `值` 是当前索引对应的元素。
- 索引和值都是**局部变量**，只在循环体内有效。
- 遍历是**有序**的，按照索引从小到大的顺序。

> [!Info]+ 信息
> - 你可以给变量起任何名字，比如 `index, value` 或 `i, v` 都可以，`ipairs` 只负责按顺序取出索引和对应的值。
> - 如果表中有空洞（`nil`），`ipairs` 会在第一个空洞处停止。

**示例 1**：简单遍历
```lua
local colors = {"红色", "绿色", "蓝色"}

for i, color in ipairs(colors) do
    print("颜色 " .. i .. "：" .. color)
end

--[[ 输出
颜色 1：红色
颜色 2：绿色
颜色 3：蓝色
]]
```

**示例 2**：遍历技能列表
```lua
local skills = {
    {name = "火球术", damage = 30, mana_cost = 10},
    {name = "治疗术", heal = 50, mana_cost = 15},
    {name = "冰箭术", damage = 25, mana_cost = 8, slow = true}
}

print("技能列表:")
for i, skill in ipairs(skills) do
    print("  技能 " .. i .. ": " .. skill.name)
    
    if skill.damage then
        print("    伤害: " .. skill.damage)
    end
    
    if skill.heal then
        print("    治疗: " .. skill.heal)
    end
    
    print("    消耗法力: " .. skill.mana_cost)
end

--[[ 输出
技能列表:
  技能 1: 火球术
    伤害: 30
    消耗法力: 10
  技能 2: 治疗术
    治疗: 50
    消耗法力: 15
  技能 3: 冰箭术
    伤害: 25
    消耗法力: 8
]]
```

### 4.5.2 pairs - 遍历所有元素

`pairs` 遍历表中的**所有键值对**（包括数组部分和字典部分），但**遍历顺序不确定**。

**语法**：

```lua
for 键, 值 in pairs(表) do
    -- 循环体
end
```
- `键` 是当前键名（可以是数字、字符串、布尔值等）
- `值` 是对应的值
- 键和值都是**局部变量**，只在循环体内有效
- **遍历顺序不保证**，每次运行可能不同

```lua
local student = {
    "小明",					-- 数组部分，索引 1
    age = 18,				-- 字典部分
    grade = "高三",			-- 字典部分
    "男",					-- 数组部分，索引 2
    ["出生日期"] = "2005-03-15"  -- 字典部分
}

for key, value in pairs(student) do
    print("  " .. key .. ": " .. value)
end

--[[ 输出
  1: 小明
  age: 18
  2: 男
  出生日期: 2005-03-15
  grade: 高三
]]
```

**示例**：遍历物品栏
```lua
local inventory = {
    gold = 150,
    silver = 300,
    ["生命药水"] = 5,
    ["魔法药水"] = 3,
    weapons = {"木剑", "铁盾"}
}

print("物品栏内容:")

for item, amount in pairs(inventory) do
    if type(amount) == "table" then
        print("  " .. item .. ":")
        
        for i, weapon in ipairs(amount) do
            print("    " .. i .. ". " .. weapon)
        end
    else
    	print("  " .. item .. ": " .. amount)
    end
end

--[[ 输出
物品栏内容:
  生命药水: 5
  魔法药水: 3
  weapons:
    1. 木剑
    2. 铁盾
  gold: 150
  silver: 300
]]
```

### 4.5.3 ipairs 和 pairs 的原理简析

你可能好奇，`ipairs` 和 `pairs` 内部到底是怎么工作的？其实，Lua 的泛型 `for` 循环需要三个东西：
1. **迭代器函数**：每次调用返回下一个值
2. **状态表**：要遍历的表（固定不变）
3. **控制变量**：当前的索引或键

`ipairs` 的工作原理可以简化为：

```lua
-- 手动模拟 ipairs 的遍历过程
local t = {"苹果", "香蕉", "橙子"}

local iter = function(state, index)
    index = index + 1
    local value = state[index]
    
    if value then
        return index, value
    end
end

local state = t   -- 状态表就是 t 本身
local control = 0 -- 初始索引 0

-- 第一次调用
local i, v = iter(state, control)  -- i = 1, v = "苹果"
while i do
    print(i, v)
    i, v = iter(state, i)          -- 用上一次的 i 作为新 control
end
```

`pairs` 的原理类似，但它使用 Lua 内置的 `next` 函数作为迭代器：

```lua
-- pairs 等价于：
for k, v in next, t, nil do
    print(k, v)
end
```

`next(t, key)` 会返回表中的下一个键值对。当 `key` 为 `nil` 时，返回第一个键值对；当没有更多键时，返回 `nil`。

**理解这个原理的好处**：
- 你可以自己写简单的无状态迭代器（比如遍历数组的 `my_ipairs`）
- 知道为什么修改被遍历的表可能导致不可预测的结果
- 明白为什么 `pairs` 的顺序不确定（`next` 的遍历顺序由表的内部实现决定）

> [!Tip]+ 提示
> 通常不需要自己实现这些迭代器，但了解原理能帮助更好地使用它们。

## 4.6 表相关函数

Lua 提供了几个内置函数来方便操作表。

### 4.6.1 table.insert - 插入元素到数组

在数组的指定位置插入元素，**后续元素自动后移**。

```lua
table.insert(表, 元素)			-- 在末尾插入
table.insert(表, 位置, 元素)		-- 在指定位置插入
```

**示例**：
```lua
local inventory = {}

-- 添加元素到数组末尾
print("添加元素到数组末尾")
table.insert(inventory, "药水")
table.insert(inventory, "钥匙")
inventory[#inventory + 1] = "地图"   -- 等价于在末尾插入

-- 修改元素
inventory.gold = 150 -- 修改现有元素
inventory[2] = "万能钥匙" -- 修改数组部分

print("库存:")
for k, v in pairs(inventory) do
    print(k .. ": " .. v)
end

print("\n插入元素 “剑” 到数组第二个索引，将元素 2 后移。")
table.insert(inventory, 2, "剑")

print("库存:")
for k, v in pairs(inventory) do
    print(k .. ": " .. v)
end
--[[ 输出
插入元素到数组末尾
库存:
1: 药水
2: 万能钥匙
3: 地图
gold: 150

添加元素 “剑” 到数组第二个索引，将元素 2 后移
库存:
1: 药水
2: 剑
3: 万能钥匙
4: 地图
gold: 150
--]]
```

### 4.6.2 table.remove - 移除数组元素

移除数组中指定位置的元素，**后续元素自动前移**。

```lua
table.remove(表, 位置?) -> 被移除的元素
```
- 不指定位置时，默认移除末尾元素。

**示例**：
```lua
local items = {
    "剑",
    "盾",
    "药水",
    "钥匙",
    "地图",
    gold = 100,
    weight = 50
}

local removed = table.remove(items, 2)	-- 移除第二个索引的元素（盾）

print("移除了" .. removed .. "，现在第二个索引的元素:" .. items[2])

print("ipairs 遍历（table.remove 自动补洞）:")
for i, v in ipairs(items) do
    print(i .. ": " .. v)
end
--[[ 输出
移除了盾，现在第二个索引的元素: 药水

ipairs 遍历（table.remove 自动补洞）:
1: 剑
2: 药水
3: 钥匙
4: 地图
]]
```

#### 为什么使用 table.remove？

如果通过赋值 `nil` 直接删除元素，删除数组元素后数组会有 “洞”会影响 `ipairs` （会在遇到第一个 `nil` 停止），`table.remove` 会自动重新排列数组。

```lua
local items = {
    "剑",
    "盾",
    "药水",
    "钥匙",
    "地图",
    gold = 100,
    weight = 50
}

print("原始物品:")
for k, v in pairs(items) do
    print(k .. ": " .. v)
end

-- 删除数组元素
items[2] = nil -- 删除第二个元素（盾）
print("\n删除盾牌后:")
for k, v in pairs(items) do
    print(k .. ": " .. v)
end

-- 注意：设置 nil 不会自动调整数组
-- 数组部分可能会有"洞"
print("\n有洞的数组遍历（ipairs会在第一个nil停止）:")
for i, v in ipairs(items) do
    print(i .. ": " .. v)
end

--[[ 输出
原始物品:
1: 剑
2: 盾
3: 药水
4: 钥匙
5: 地图
weight: 50
gold: 100

删除盾牌后:
1: 剑
3: 药水
4: 钥匙
5: 地图
weight: 50
gold: 100

有洞的数组遍历（ipairs会在第一个 nil 停止）:
1: 剑
```

### 4.6.3 table.concat - 连接为字符串

将数组部分连接成一个字符串，可以指定分隔符和范围。

```lua
table.concat(表, 分隔符?, 起始索引?, 结束索引?)
```

**示例**：
```lua
local words = {"Hello", "Lua", "World", "!"}

print(table.concat(words))          --> HelloLuaWorld!
print(table.concat(words, " "))     --> Hello Lua World !
print(table.concat(words, ", ", 2, 3)) --> Lua, World

-- 构建文件路径
local path_parts = {"home", "user", "projects", "main.lua"}
local path = table.concat(path_parts, "/")
print("路径: " .. path)   --> home/user/projects/main.lua
```


### 3.6.4 `table.sort` - 排序

对数组部分进行排序，可以指定自定义比较函数。

```lua
table.sort(表, 比较函数?)
```
- 不指定函数时，根据数字升序排序。

> [!Info]+ 信息
> 比较函数参数应该有两个，返回布尔值。返回 `true` 时表示 a 应该在 b 前面。

**示例 1**：按数字大小排序
```lua
local numbers = {5, 2, 8, 1, 9, 3}
-- 按数字大小升序排序
table.sort(numbers)
print(table.concat(numbers, ", "))   --> 1, 2, 3, 5, 8, 9

-- 降序
-- a 如果大于 b，a 排在 b 前面
table.sort(numbers, function(a, b)
    return a > b
end)
print(table.concat(numbers, ", "))   --> 9, 8, 5, 3, 2, 1
```

**示例 2**：复杂数据排序（按血量降序，血量相同按等级降序）
```lua
local players = {
    {name = "alice", health = 85, level = 18},
    {name = "mike", health = 92, level = 17},
    {name = "tom", health = 78, level = 19},
    {name = "jerray", health = 85, level = 20}
}

table.sort(players, function(a, b)
	-- 血量相同按等级降序
    if a.health == b.health then
        return a.level > b.level
    end
    
    -- 血量不相同按血量降序
    return a.health > b.health
end)

for i, p in ipairs(players) do
    print(i .. ". " .. p.name .. " (血量：" .. p.health .. "，等级：" .. p.level .. ")")
end
--[[ 输出
1. mike (血量：92，等级：17)
```

## 4.7 表的复制

表的复制原理是将值类型提取出来放到一个新表中。

### 4.7.1 浅复制 - 只复制第一层，嵌套的表仍是引用

```lua
local function shallow_copy(original)
    local copy = {}
    
    for k, v in pairs(original) do
        copy[k] = v
    end
    
    return copy
end

local original = {a = 1, b = {x = 10}}
local copy = shallow_copy(original)
copy.a = 100
print(original.a)		--> 1（完全独立）

original.b.x = 999
print(copy.b.x)   --> 999（因为 b 是同一个表）
```

### 4.7.2 深复制 - 完全独立，递归复制所有嵌套表

```lua
-- 简化版（不考虑循环引用与复杂键）
local function deep_copy(original)
    if type(original) ~= "table" then
        return original
    end

    local copy = {}

    for k, v in pairs(original) do
        copy[k] = deep_copy(v)
    end

    return copy
end

local original = {a = 1, b = {x = 10}}
local deep = deep_copy(original)
original.b.x = 888

print(deep.b.x)   --> 10（独立副本，不受影响）
```

## 4.8 表的合并

原理同上。

#### 4.8.1 浅合并 - 后覆盖前，嵌套表仍是引用

```lua
-- 合并两个表（浅合并）
local function shallow_merge(t1, t2)
    local result = {}
    
    for k, v in pairs(t1) do
        result[k] = v
    end
    
    for k, v in pairs(t2) do
        result[k] = v
    end
    
    return result
end
```

#### 4.8.2 深合并 - 递归合并嵌套表

```lua
-- 简化版（不考虑循环引用与复杂键）
local function deep_merge(t1, t2)
    local result = {}
    
    for k, v in pairs(t1) do
        result[k] = v
    end
    
    for k, v in pairs(t2) do
    	-- 被合并的值与合并的值为表时进一步递归合并
        if type(result[k]) == "table" and type(v) == "table" then
            result[k] = deep_merge(result[k], v)
        else
            result[k] = v
        end
    end
    
    return result
end
```

## 4.9 冒号语法糖

表可以存储函数，Lua 提供了**冒号语法糖**，让表访问更自然。

```lua
local player = {
    name = "英雄",
    health = 100,
    level = 1
}

-- 定义方法（使用冒号自动添加 self 参数）
function player:take_damage(amount)
    self.health = self.health - amount
    
    if self.health <= 0 then
        self.health = 0
        print(self.name .. " 倒下了！")
    else
        print(self.name .. " 受到 " .. amount .. " 伤害，剩余生命: " .. self.health)
    end
end

function player:heal(amount)
    self.health = self.health + amount
    print(self.name .. " 恢复了 " .. amount .. " 生命，当前生命: " .. self.health)
end

-- 调用方法（冒号会自动把调用者作为第一个参数传入）
player:take_damage(30)   -- 英雄 受到 30 伤害，剩余生命: 70
player:heal(10)          -- 英雄 恢复了 10 生命，当前生命: 80
```

> [!Info]+ 信息
> ```lua
> -- 定义时
> function player:method(arg) ... end
> -- 等价于
> function player.method(self, arg) ... end
>
> -- 调用时
> player:method(arg)
> -- 等价于
> player.method(player, arg)
> ```
